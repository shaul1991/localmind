# OQ-V2 검색 스택 실험 — 임베딩 vs 구조 검색

> **TL;DR** — 실쿼리 20개에서 **A(임베딩) 6승 · B(구조 검색) 6승 · 무승부 8** — 완전 동률.
> 단, 승리 프로필이 갈린다: **B는 "결정 노트 소환"(vision의 핵심 목적)에서 이기고, A는
> "과거 세션 고고학"(옛 목적)에서 이긴다.** 좁아진 목적 기준으로는 **임베딩 제거 권고**
> (동률이면 더 싼 쪽 + 목적 적합 쪽). 최종 결정은 사용자 게이트.

## 1. 방법

- **쿼리셋**: 서버 query-log 백업(87건, `query-log-homeserver-20260721.jsonl`) + 로컬
  `~/.localmind/query-log.jsonl`(42건)에서 search_notes 쿼리 추출·중복 제거 후 **대표 20개**
  선정. 선정 기준: 의도 클러스터별 대표(빈도 가중 — "Source of Truth" 21회는 1개로,
  PAP/paperclip 근중복 ~10건은 3개로), 한/영·결정회상/문서조회/탐색 유형 커버.
- **방법 A(임베딩)**: 프로덕션 경로 그대로 — `searchNotes()`(bge-m3·cosine-full-scan) 직접
  호출. `QUERY_LOG`를 스크래치로 격리(프로덕션 로그 오염 방지).
- **방법 B(구조 검색)**: 신규 의존성 0의 ~40줄 스코어러 — 토큰별 제목 매치 ×3 + frontmatter
  태그 ×2 + 본문 빈도 ×1(cap 5) + 최근성 부스트(+2×(1−age/365)). 코퍼스 1273 md 파일
  (NOTES_DIR 3루트 — A의 색인 1240파일과 근사 동일).
- **평가**: 쿼리별 양쪽 top-5를 나란히 두고 "쿼리 의도에 더 잘 답하는 쪽"을 judge(LLM 단일)
  판정, 무승부 허용.

## 3. 패턴 분석 (집계보다 중요)

1. **잘 지어진 결정 노트 제목 앞에서 임베딩은 이점이 없다** — 정답이 결정 노트인 쿼리
   (10·13·14·15)는 전부 무승부(둘 다 top1 정답). localmind의 capture는 제목에 "결정: …"과
   태그를 구조적으로 남기므로, **vision이 좁힌 목적(why 소환)에서는 제목×태그×최근성만으로
   충분**하다는 실증.
2. **B의 승리는 전부 "결정 회상" 쿼리**(2·4·5·11 — 최근성 부스트가 최신 결정을 끌어올림),
   **A의 승리는 전부 "옛 세션 고고학·의미 패러프레이즈" 쿼리**(1·7·9·12·16·18 — 쿼리 단어와
   문서 단어가 다를 때). 좁아진 목적 기준으로 B의 승리 프로필이 목적 정합.
3. **A의 관찰된 결함**: 동일 청크 중복 반환(11·19 — top5 중 4개가 같은 내용), 낡은 대형 세션
   캡처가 최신 결정을 가림(2·4). 이것이 사용자 체감 "나오는데 원하는 게 아님"과 일치.
4. **양쪽 공통 실패 2건**(3·6): 정답이 어휘가 다른 기기 노트에 있음 — 검색 알고리즘이 아니라
   **주입(브리핑) 계층이 풀 문제**(vision §6-3)라는 방증.

## 4. 판정 권고안

**권고: 임베딩 제거(구조 검색 전환)** — 확신도 중간.

- 근거: ① 동률이면 더 싼 쪽(Ollama 의존·~13분 재색인·색인 파일·설치 마찰 제거 — 비개발자
  접근성 급증) ② 좁아진 목적(결정 소환)에서 B의 승리 프로필이 정합 ③ A의 결함(중복 청크·
  낡은 캡처 우위)은 chunk-cosine 구조 내재적, B의 결함(키워드 수프)은 파일 중복 제거·필드
  가중 조정으로 싸게 개선 가능.
- 제거 시 감량 목록(집행은 great-reduction 슬라이스 소관): Ollama/EMBEDDINGS_* 의존,
  reindex 파이프라인(스크립트·make 타깃·~13분 색인), `.brain-index.json`, brain.ts의 임베딩
  호출·청크·cosine 경로, retrieval-quality의 임베딩 전제 부분.
- **뒤집을 신호**(명시): 전환 후 "원하는 노트를 못 찾는" 체감 실패가 누적 관찰되거나
  query-log 0-히트율이 상승하면 임베딩 복원(코드는 git 역사에 보존 — 비가역 아님). 특히
  패러프레이즈 질의가 주 사용 패턴이 되면 재평가.
- **최종 결정은 사용자 게이트다** — 이 문서는 권고까지만.
