# Goal: 원격 MCP 접속(HTTP) — 단일 두뇌를 여러 기기에서 공유

<!-- 왜(why) 이 작업이 존재하는가. 구현 세부(스키마·API·파일경로·구현 단계)는 넣지 않는다. -->

## Background — 배경
localmind는 지금 MCP를 **stdio 전용**으로 노출한다(`src/mcp.ts` → `StdioServerTransport`).
즉 호스트 앱이 `node dist/mcp.js`를 **같은 머신에서 로컬 서브프로세스로** 띄우는 방식이라,
두뇌(노트·기억 저장소)가 기기마다 로컬에 있고 기기 간 공유는 git 기반 동기화(specs/019·031)에
의존한다. 스택 서버(:8787)도 설계상 루프백 전용(`127.0.0.1`)이다.

여러 기기(항상 켜진 서버 1대 + 노트북 여러 대)에서 **하나의 두뇌**를 쓰고 싶을 때,
현재 구조는 기기마다 복사본 + 동기화라 지연·충돌 여지가 있다.

## Problem — 문제
설치한 개인이 기기를 여러 대 쓸 때(예: 항상 켜진 리눅스 홈서버 + 노트북 여러 대),
- 각 기기가 자기 로컬 두뇌를 갖고 git으로 동기화 → **실시간이 아니고**, push/pull 지연·충돌이 생긴다.
- 한 기기에서 `capture_note` 한 내용이 다른 기기에서 즉시 검색되지 않는다.
- MCP가 stdio라 **원격 기기에서 중앙 두뇌에 접속할 수단 자체가 없다**.

## Objective — 목표
localmind MCP를 **네트워크로 접속 가능한 HTTP(Streamable HTTP) 전송 모드**로도 띄울 수 있게 해,
항상 켜진 호스트 1대가 **단일 두뇌**를 서비스하고 다른 기기의 MCP 클라이언트가 그 하나에
붙게 한다. 단일 원본(single source of truth)이 되면 "동기화"라는 문제 자체가 사라진다.

## Expected outcome — 기대 결과
- 홈서버(ubuntu)에서 `make mcp-serve-http`로 인증이 걸린 HTTP MCP 엔드포인트를 띄운다.
- 노트북(맥)의 Claude Code가 `claude mcp add --transport http`로 그 하나에 붙어, 어느 기기에서
  적재해도 즉시 다른 기기에서 검색·회상된다(중앙 저장소이므로 동기화 불필요).
- 기존 stdio 사용자는 **무엇도 바뀌지 않는다**(기본값 유지).

## Non-goals — 비목표
- **iPhone(Claude 모바일 앱)·공용 인터넷 노출**: 이번 범위 제외. 모바일 앱 커넥터는 Anthropic
  클라우드에서 접속 → 공용 HTTPS URL + 강인증(터널)이 필요해 별도 스펙으로 분리한다.
  이번엔 **맥은 Tailscale 사설망**까지만(사용자 결정, 2026-07-12).
- Claude Desktop 클라우드 커넥터 UI 지원(클라우드 경유라 사설 URL 불가) — 필요 시 로컬
  `mcp-remote` 프록시로 우회하는 방법만 문서로 안내.
- 멀티테넌시·사용자별 격리(단일 사용자 전제 유지).
- 임베딩 성능 최적화·GPU 도입(임베딩은 CPU로 감당, 별도 관심사).
- TLS 종단 자체 구현(터널/리버스 프록시 또는 Tailscale이 담당 — 이번엔 평문 위에 토큰 인증 +
  사설망 전제, 공용 노출 스펙에서 TLS 강제).

## Constraints — 제약
- **하위호환 절대**: 기본(무설정) 동작은 stdio 그대로. `make mcp-install`/`mcp-desktop` 무영향.
- **최소 보안**: HTTP 모드는 Bearer 토큰 인증을 **강제**한다(토큰 없으면 기동 거부). 개인 두뇌를
  네트워크에 노출하므로 인증 없는 노출은 결함으로 본다.
- **사설 우선 바인딩**: 기본 바인드는 비공개(localhost). 네트워크 노출은 명시적 opt-in
  (`MCP_HTTP_HOST`)으로만. 실수로 0.0.0.0 공개되지 않게 한다.
- SDK: `@modelcontextprotocol/sdk` `^1.29.0`. 이미 의존 중인 express 재사용.
- 비개발자도 사용자 — 에러 메시지·도구 응답은 평이한 한국어, 절대경로 플레이스홀더.

## Stakeholders — 이해관계자
- 단일 사용자(설치한 개인 누구나 — 비개발자 포함). 특히 기기를 여러 대 쓰는 사용자.

## Risks — 리스크
- **노출 사고**: 바인드/방화벽 실수로 두뇌가 의도보다 넓게 노출될 수 있다 → 기본 비공개 +
  토큰 강제 + 문서로 완화.
- **토큰 유출**: Bearer 토큰이 곧 두뇌 접근권. 문서에서 토큰 관리·회수 절차를 안내.
- **SDK API 편차**: 1.29.x의 Streamable HTTP 전송 클래스/시그니처를 구현 시점에 설치본으로
  재확인(아래 plan Open questions).
- **단일 호스트 SPOF**: 중앙화하면 호스트가 꺼지면 전 기기가 두뇌를 잃는다 → 항상 켜진
  서버를 호스트로 두는 선택으로 완화(사용자 결정: ubuntu).
