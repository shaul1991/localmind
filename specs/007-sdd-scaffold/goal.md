# Goal: SDD Scaffold (일관된 작업 흐름을 어디서든 심기)

## Background — 배경

ROADMAP.md의 비전이 확장됐다: "누구든 어느 기기·어느 AI 도구를 쓰든 localmind를 통해 같은
SDD 작업 흐름(goal→spec→plan→구현→self-review)과 규율을 설정할 수 있어야 한다." 지금은
이 흐름(AGENTS.md의 SDD 규칙 + goal/spec/plan 문서 템플릿)이 localmind 저장소 자체에만
존재한다 — 다른 프로젝트에 적용하려면 사용자가 파일을 수동으로 복사·수정해야 한다.

## Problem — 문제

- 새 프로젝트를 시작하거나 새 기기에서 작업할 때, SDD 흐름(AGENTS.md 규칙 + 문서 템플릿)을
  매번 손으로 재현해야 한다.
- AI 도구(Claude Code·Codex·MCP 호환 도구 등)나 기기가 달라져도 일관되게 적용할 표준
  배포 수단이 없다.
- localmind 자체의 AGENTS.md는 localmind 전용 내용(오픈소스 절 등)이 섞여 있어 그대로
  다른 프로젝트에 복사하기에 적합하지 않다.

## Objective — 목표

localmind가 **범용화된 AGENTS.md 규칙 + goal/spec/plan 문서 템플릿**을 사용자가 지정한
대상 디렉토리(localmind 밖의 어떤 프로젝트든)에 심을 수 있게 한다. 이 기능은 **MCP 도구**와
**make 커맨드** 두 경로 모두로 제공한다 — 어느 한쪽에 의존하면 "어느 AI 도구든"이라는
목표를 충족할 수 없기 때문이다(MCP 미지원 도구·MCP 연결이 불안정한 환경 대비).

## Expected outcome — 기대 결과

- `make init-sdd DIR=<대상경로>`를 실행하면 해당 경로에 범용 `AGENTS.md`와 `specs/` 폴더가
  생성된다.
- MCP 연결된 AI 도구가 `scaffold_sdd` 같은 도구를 `path` 파라미터로 호출해도 동일한 결과가
  생성된다.
- 두 경로 모두 **같은 정본 템플릿**을 참조한다 — 템플릿이 두 곳에 따로 유지되며 드리프트되지
  않는다.
- 생성되는 AGENTS.md에는 localmind 전용 내용이 섞이지 않는다(순수 SDD 흐름 규칙만).
- 대상 경로에 이미 관례 파일(AGENTS.md 등)이 있으면 덮어쓰지 않고 안전하게 알린다.

## Success metrics — 성공 지표

- make 커맨드와 MCP 도구가 동일한 입력에 대해 동일한 결과물을 생성함(정본 템플릿 재사용 확인).
- 생성된 AGENTS.md 템플릿을 검사했을 때 localmind 고유 문구·명령어가 0건.
- 기존 파일이 있는 대상 경로에 실행해도 기존 파일이 손상되지 않음(회귀 없음).

## Non-goals — 비목표

- GitHub `spec-kit`의 `specify` CLI를 대체하는 것이 아니다 — `specify`는 spec/plan 세부
  작성을 돕는 별도 도구이며, 이 기능은 "규칙 + 최소 문서 골격 배포"에 집중한다. 상호 보완
  관계이며 경쟁하지 않는다.
- 대상 프로젝트에 이미 있는 AGENTS.md/CLAUDE.md를 자동으로 병합·수정하는 것은 범위 밖 —
  충돌 시 건너뛰고 사용자에게 알리기만 한다.
- 원격/클라우드 배포는 범위 밖 — 로컬 파일시스템에 파일을 쓰는 것으로 한정한다.
- `specs/`에 예시 feature(001 샘플)를 채워 넣는 것은 범위 밖 — 빈 폴더 구조만 만든다.

## Constraints — 제약

- 로컬 전용 — 원격 서버·네트워크 호출 없이 파일시스템 복사로만 동작한다(기존 아키텍처 원칙 유지).
- 템플릿은 localmind 저장소 안에 **정본 하나**로 관리하고, MCP 도구와 make 커맨드 둘 다 같은
  정본(같은 함수/같은 템플릿 파일)을 참조한다 — 로직 중복 금지.
- 기존 파일을 덮어쓰지 않는다(안전 우선 — "사람이 직접 확인 후 병합" 원칙, `architecture-
  constitution`의 "적용은 사람 주도가 기본" 정신과 동일).
- 에러·결과 메시지는 비개발자가 이해할 수 있는 평이한 한국어로 작성한다.
- TypeScript, 기존 `src/` 모듈 패턴(stdout 미사용 등 MCP 규약) 유지.

## Stakeholders — 이해관계자

- 여러 기기·여러 프로젝트·여러 AI 도구에서 일관된 SDD 작업 흐름을 원하는 localmind 사용자
  누구나(설치한 개인 — 비개발자 포함)

## Risks — 리스크

- **기존 관례와 충돌**: 대상 프로젝트에 이미 다른 내용의 AGENTS.md/CLAUDE.md가 있으면
  스캐폴드가 무의미하거나 혼란을 줄 수 있음 — 덮어쓰지 않고 명확히 알리는 것으로 완화.
- **오용 위험**: MCP 도구가 파일시스템 쓰기 권한을 가지므로, 경로 파라미터가 없거나
  잘못되면 의도치 않은 위치에 쓸 위험 — `path` 파라미터를 필수화하고 검증한다.
- **템플릿 드리프트**: localmind 자체의 AGENTS.md(프로젝트 전용)와 배포용 범용 템플릿이
  서로 다른 파일로 존재하므로, 시간이 지나며 내용이 벌어질 수 있음 — 문서화로 인지시키고
  공통 부분은 가능한 한 구조를 맞춘다.
