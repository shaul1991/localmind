# 사용법 — 매일 이렇게 씁니다

설치를 마쳤다면([README Quickstart](../README.md#시작하기)), 이제 **그냥 채팅창에서 말로** 씁니다.
Claude Desktop / Cursor에서 평소처럼 대화하되, 아래 흐름을 자연어로 시키면 됩니다.

> 개념이 처음이면 👉 [비유로 이해하기](concepts.md) · 궁금증은 👉 [FAQ](faq.md)

---

## 한눈에 — 네 가지 흐름

```
① 쌓기(캡처)      →  ② 찾기(검색)       →  ③ 확인(식별)    →  ④ 이어가기(브리핑)
   이거 기록해둬        그거 뭐였지·찾아줘      지금 어떤 두뇌지?     이 프로젝트 결정 알려줘
```

전부 **채팅창 안에서 말로** 끝납니다. 별도 화면·터미널 필요 없습니다.

## ① 쌓기 — 노트로 남기기

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 결정·지식·맥락 기록 | "이 내용(결정·이유) **노트로 저장**해줘" | `capture_note` |
| **결정을 구조로 기록** | "이 **결정**(선택·이유·전제까지) 저장해줘" | `capture_note` (choice·why·assumptions) |

- **노트(.md)** = `NOTES_DIR`(기본 `~/.localmind`)의 마크다운 파일이 정본.
- 가장 값진 기록은 **왜 그렇게 결정했는지(why)** — AI가 대화의 부산물로 알아서 남기게 하세요.
- **결정 캡처의 3층**: 선택(무엇을 골랐나)·이유(왜 — 불변의 자산)·전제(당시의 사실 —
  `volatility: high`면 시간이 지나 낡을 수 있는 층). 전제마다 검증 시각이 자동 기록되고,
  나중에 낡으면 검색·브리핑에서 알려줍니다(아래 ④).

## ② 찾기 — 꺼내 쓰기

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 지난 결정·지식 찾기 | "결제 **뭐로 정했지**? 노트에서 찾아줘" | `search_notes` |
| 노트 근거로 답 받기 | "내 노트 기준 우리 원칙이 뭐야?" | `search_notes` → 호스트 AI가 종합 |

→ 검색은 의미 기반입니다(단어가 정확히 같지 않아도 찾아줍니다). 답의 종합·해석은
호스트 AI(Claude 등)가 검색 결과를 근거로 수행합니다.

## ③ 확인 — 지금 어떤 두뇌인지

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 기기·노트 폴더 확인 | "whoami" | `whoami` |

## ④ 이어가기 — 세션 시작 브리핑과 살아있는 결정

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 과거 결정 갖고 시작 | "이 프로젝트 관련 **결정 브리핑**해줘" | `brief` |

- **자동 연결(권장)**: 쓰는 AI 도구의 지침 파일(`CLAUDE.md`·`AGENTS.md` 등)에 한 줄 —
  `새 세션 시작 시, 작업 주제로 localmind의 brief 도구를 호출해 과거 결정을 확인한다.`
  이러면 매번 부탁하지 않아도 새 세션이 결정을 가지고 시작합니다.
- **낡음 신호(⏳)**: 검색·브리핑 결과에 전제가 오래 미검증(`BRIEF_STALE_DAYS`, 기본 30일)인
  결정이 있으면 응답 끝에 한 줄 알림이 붙습니다 — 흐름을 막지 않는 신호일 뿐이니 무시해도
  됩니다. 확인했다면 AI에게 "최신 사실 확인하고 그 노트의 last_verified 갱신해줘"라고 하면
  됩니다(노트는 평문 파일이라 직접 수정으로 끝 — 다음 조회부터 신호가 사라집니다).

## 정리 — 노트는 파일로 직접

노트의 열람·수정·삭제는 **파일시스템/에디터에서 직접** 합니다(정본이 파일이므로) — 변경은
재인덱싱이 따라갑니다(`make reindex`, 저장 시 자동 감지도 동작). 휴지통 확인 `make trash-list`,
완전 비우기(비가역)는 `make trash-empty`.

## 도구를 바꿔 써도 — 같은 두뇌

Cursor에서 기록한 걸 Claude Desktop에서 꺼내 씁니다. 전제: 각 도구의 MCP 설정에서
**같은 `NOTES_DIR`** 를 쓰면 됩니다(= 같은 두뇌 공유). 단, *저장된* 것만 공유됩니다 —
중요하면 "노트로 기록해둬"로 박아두세요.

## 여러 주제로 나누고 싶다면 (선택)

기본은 폴더 하나(`~/.localmind`)로 충분합니다. 주제별로 나누고 싶으면:
```
NOTES_DIR="work=/내/업무노트,life=/내/개인노트"
```
→ `search_notes`에 `folder:"work"` 로 한정할 수 있습니다.

## 백업 · 새 기기 복구

> 두 기기를 함께 쓴다면: `make device-sync HOST=<라벨>` 한 명령으로 다른 기기를 최신화할 수
> 있어요(CI 확인→백업→원격 pull→검증 — `.env`의 `SYNC_DEVICES` 설정, 새 기기는 `make recover` 먼저).

- **백업**: `make backup` (노트·자산을 내 private git repo로). 자동은 `make backup-cron`.
- **복구**: 새 컴퓨터에서 `make recover RESTORE_REPO=<내 백업 repo>` 한 줄.
  노트 저장소를 여러 개 쓰면 recover 후 `make notes-connect` → `make restore` 한 번 더(recover가 안내).
- **검색 기록(선택)**: 기본은 로컬 전용 — `BACKUP_QUERY_LOG=1 make backup`으로 기기 간
  병합 가능(검색어가 git 이력에 남는 점 주의 — [FAQ 참조](faq.md)).
- 자세히 👉 [백업·복구 문서](backup.md) · [FAQ 백업 절](faq.md#5-운영--성능과-백업)

## 막히면

- 도구가 안 보임 → **Claude Desktop/Cursor 재시작**(MCP는 시작 시 로드)
- 재색인 후 "보존 중이에요" 안내가 보임 → 지금 설정에 없는 폴더의 색인을 지키고 있다는
  뜻(지워진 게 아님). 안 쓰는 폴더면 안내된 `REINDEX_PRUNE_LABELS=<라벨> make reindex`로
  정리 — `make doctor`의 [색인 라벨] 섹션에서도 확인 가능 👉 [FAQ](faq.md#5-운영--성능과-백업)
- 더 많은 증상 👉 [문제 해결 문서](troubleshooting.md)
