# Goal: 노트 카드 브라우저 — 적재 내용을 웹에서 카드로 훑기

<!-- 왜(why) 이 작업이 존재하는가. 구현 세부(스키마·API·파일경로·구현 단계)는 넣지 않는다. -->

## Background — 배경

034에서 localmind **상태**를 보는 모니터링 웹 UI(사이드바+카드 그리드 SPA, 라이트/다크,
디자인 토큰)를 만들었다. 하지만 정작 **적재된 노트 내용 자체**를 브라우저에서 볼 방법은 없다 —
지금은 MCP `recall`/`search_notes`나 파일을 직접 열어야 한다.

## Problem — 문제

- localmind에 무엇이 쌓였는지(캡처·결정 노트·지식 문서)를 **한눈에 훑어볼 수 없다.**
- 제목만 나열되는 목록으로는 "무슨 내용인지" 감이 안 온다 — 스니펫·태그·날짜가 함께 보여야 한다.
- 비개발자도 자기 세컨드 브레인을 브라우저에서 편하게 둘러보고 싶다.

## Objective — 목표

034 웹 UI에 **노트 카드 브라우저 페이지(`#/notes`)**를 추가한다. 적재된 **전체 노트를 카드
그리드**(제목·폴더·태그·날짜·스니펫)로 보여주고, **검색·태그 필터**로 좁히고, 카드를 열면
**본문을 읽을 수 있게** 한다. 기존 디자인 시스템(034)·읽기전용·로컬 전용을 그대로 계승한다.

## Expected outcome — 기대 결과

- 사이드바에 "노트" 메뉴 → 카드 그리드로 전체 노트가 보인다.
- 검색어·태그로 즉시 필터된다(예: `decision` 태그로 결정 노트만).
- 카드를 클릭하면 노트 본문을 읽을 수 있다.
- 034의 토큰·상태전이(로딩/빈/에러)·라이트/다크를 그대로 따른다.

## Success metrics — 성공 지표

     미달성은 체크하지 않고 사유 부기(은폐 금지). -->
- [x] `#/notes`에서 적재 노트가 카드 그리드로 렌더된다. *도그푸드: 실벌트 1089노트 카드(제목·폴더·태그·날짜·스니펫).*
- [x] 검색어·태그로 즉시 필터된다. *도그푸드: 검색 1089→7, 태그 design→1.*
- [x] 카드를 열면 본문을 읽을 수 있다(경로 안전). *도그푸드 리더 + 경로안전 테스트(트래버설·심링크 거부).*
- [x] 대량 노트(1000+)에서도 UI가 멈추지 않는다. *도그푸드: 1089노트 60장 렌더 + "더 보기".*
- [~] 기존 034 회귀 0·키게이트 유지·라이트/다크 정상. *397 green·라이트 도그푸드 O / **다크 시각 미검증**(토큰 구조적 커버).*

## Non-goals — 비목표
<!-- 이번 범위에서 제외하는 것. (필수) -->
- **노트 편집·삭제·생성** — 읽기 전용(034 계승). 쓰기는 MCP `capture`가 담당.
- **의미 검색(임베딩)** — v1은 제목·스니펫·태그 **텍스트 필터**만(recall은 MCP가 담당).
- **그래프 뷰·백링크 시각화** — 후속(005 note-link-graph와 별개).
- **원격 접근·인증 개편** — 034의 로컬 전용·KeyGate 그대로.

## Constraints — 제약
<!-- 기술·운영·일정·외부 의존 제약. (필수) -->
- **034 디자인 시스템 준수**: 새 토큰·프레임워크 금지. 기존 CSS 변수·`el/badge/card/api` 재사용.
- **읽기 전용·로컬 전용·KeyGate**: 034 보안 자세 계승(127.0.0.1, 키 게이트).
- **경로 안전**: 노트 본문 읽기는 폴더 루트 밖·심링크 접근 차단(034 reportsStatus 보안 리뷰 계승).
- **빌드리스·외부요청 0**: vanilla, 폰트·CDN 금지.
- **대량 노트 성능**: 1000+ 노트에서 렌더가 브라우저를 멈추지 않게(상한/점진).

## Stakeholders — 이해관계자

- 단일 사용자(설치한 개인 — 비개발자 포함). 자기 세컨드 브레인을 브라우저로 둘러보려는 사용자.

## Risks — 리스크

- **대량 DOM**: 1066개 카드 일괄 렌더 → 멈춤. 상한/점진 렌더로 대응(설계).
- **스니펫·제목 추출 편차**: 노트마다 frontmatter/제목 형식이 달라 추출이 흔들릴 수 있음 →
  프론트matter·첫 헤딩·첫 유효줄 순 폴백.
- **민감 노트 노출**: 벌트엔 민감 노트도 있음 — 로컬 전용+KeyGate로 접근 차단(외부 노출 아님).
