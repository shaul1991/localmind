# Goal: 한국어 임베딩 A/B — qwen3-embedding:0.6b vs bge-m3

<!-- 왜(why) 이 작업이 존재하는가. 구현 세부(스키마·API·파일경로·구현 단계)는 넣지 않는다. -->

> **조사 기반·Live-Verify 주의**: 아래 모델 수치는 2026-07-06 시점 공식/집계 조사분(bge-m3
> MTEB 63.0, qwen3-embedding:0.6b 639MB·1024 dims·32K context 등, T1~T3)이다. 정확한
> dims·context·ollama 태그·한국어 서브스코어는 **구현 착수 시 라이브 재확인**한다(plan Phase 0).

## Background — 배경

localmind는 노트 임베딩에 **bge-m3**(ollama 로컬, 1024 dims, 8K context, MTEB 63.0)를 쓴다.
로컬 우선·0원·프라이버시 원칙상 임베딩은 로컬 유지가 정본이다(035 Non-goal에서 확정). 최근
Live-Verify 조사에서 **qwen3-embedding:0.6b**가 bge-m3 대비 잠재적 상위로 확인됐다: RAM 639MB
(bge-m3 ~1.2GB보다 가벼움), dims 1024(동일), context 32K(4×), MMTEB 다국어 선두 계열.

## Problem — 문제

- qwen3-0.6b가 **집계 벤치(MMTEB)에서 앞선다고 해서 "내 벌트의 한국어 회상"이 실제로 나아지는지는
  모른다.** 집계 다국어 우위 ≠ 한국어 우위(bge-m3는 원래 한국어가 강함 — MIRACL ko 71.8).
- 임베딩 모델 교체는 **전 벌트 재색인**을 강제하는 비용 큰 변경이다 — 근거 없이 바꾸면 회귀 위험.
- 즉 "바꿔야 하나?"에 답할 **실측 근거가 없다.**

## Objective — 목표

**내 실제 벌트에서 한국어 회상을 recall@k로 A/B 측정해**, qwen3-embedding:0.6b가 bge-m3를
이기는지 판정하고 **이길 때만 채택**한다. 로컬(ollama) 범위를 벗어나지 않고, 기존 bge-m3
운영 색인을 깨지 않는 **비파괴 실험**으로 수행한다. 결과가 bge-m3 우위/무의미하면 **유지**한다.

## Expected outcome — 기대 결과

- 라벨링된 한국어 질의셋 + 재현 가능한 A/B 하네스 + 두 모델의 recall@k 표.
- "채택(qwen3-0.6b) / 유지(bge-m3)" 중 하나로 **근거와 함께** 결론.
- 16GB M5에서 qwen3-0.6b가 스택과 함께 돌아가는지(RAM·지연) 실측.

## Success metrics — 성공 지표

- [ ] 두 모델 각각의 **recall@5·recall@10·MRR**이 같은 질의셋으로 산출됐다.

- [ ] 기존 bge-m3 운영 색인·설정이 **변경되지 않았다**(비파괴 검증).

## Non-goals — 비목표

- **embeddinggemma·arctic-embed2·4B/8B 등 다른 후보** — 036은 qwen3-0.6b vs bge-m3 2자 비교만.
  (embeddinggemma는 context 2K 제약 별도 검토 대상 — 후속.)
- **Gemini/클라우드 API 임베딩** — 로컬 우선 원칙 위배(035 Non-goal 계승).
- **청킹 전략 변경** — 동일 청킹에서 모델만 A/B(변수 통제).
- **운영 자동 전환** — 036은 "결정"까지. 채택 결정 시 실제 교체는 게이트 후 별도 단계.

## Constraints — 제약
<!-- 기술·운영·일정·외부 의존 제약. (필수) -->
- **로컬 전용**: 후보는 ollama 로컬 모델만(프라이버시·0원 유지).
- **16GB 예산**: 임베딩 모델이 도커 스택(챗·게이트웨이·mem0)·OS와 통합메모리 공유 → 소형만.
- **비파괴**: bge-m3 운영 색인·설정을 건드리지 않고 병렬로 실험.
- **재현성**: 질의셋·하네스·결과가 버전드로 재실행 가능.
- **Live-Verify**: 모델 dims·context·ollama 태그·한국어 성능은 기억 아닌 라이브 확인 후 사용.

## Stakeholders — 이해관계자

- 단일 사용자(설치한 개인 — 한국어 노트를 쓰는 누구나). 실행 환경: MacBook Pro M5 16GB(대표).

## Risks — 리스크

- **질의셋 편향·과소**: N이 작거나 편향되면 신호가 약함 → 최소 개수·다양성 확보(Open q).
- **qwen3 한국어가 bge-m3보다 못할 수 있음** → 그러면 **유지가 정답**(실패 아님, 근거 확보가 목적).
- **재색인 비용**: 실험용 병렬 색인도 시간 듦(8,277청크급) → 1회 한정, 비파괴.
- **판정 임계**: "이겼다"의 margin을 미리 합의 안 하면 결론이 모호 → spec Open q에서 확정.
