# Goal: 웹 설정 페이지 — 상태·복사·안내로 터미널 허들 낮추기

## Background
`make ui`(specs/034)는 읽기 전용 모니터링 웹이다. 설정 페이지가 이미 있으나 `.env` 값을
보여주고 "바꾸려면 make X"라는 문구만 줄 뿐이다. 실제 변경·설치(MCP 연결, 백엔드 추가)는
사용자가 **웹을 떠나 터미널에서 `make`를 직접 실행**해야 한다.

## Problem
비개발자에게 "웹에서 상태를 보다가 → 터미널로 나가 명령을 실행 → 다시 웹으로" 왕복은 허들이
크다. 특히 (1) **지금 뭐가 연결돼 있고 뭐가 안 됐는지** 웹에서 알 수 없고, (2) **정확히 어떤
명령을 쳐야 하는지** 외우거나 문서를 뒤져야 한다. localmind는 비개발자 포함 오픈소스라 이 마찰이
곧 진입 이탈로 이어진다.

## Objective
웹 설정 페이지에서 **각 설정의 연결/설정 상태를 한눈에 보고**, **바꿀 정확한 명령을 즉시
복사**해 따라 할 수 있게 한다 — 단 **웹은 읽기 전용을 유지**한다(설정 쓰기·실행은 여전히
터미널). "무엇이 됐나 / 무엇을 쳐야 하나" 두 허들을 새 공격면 없이 없앤다.

## Expected outcome
- 사용자가 웹에서 Claude 인증·MCP 연결(Code/Desktop)·백엔드·노트 폴더의 **현재 상태를 배지로** 확인.
- 각 항목 옆 **복사 버튼**으로 실행할 명령을 한 번에 얻어 터미널에 붙여넣기 → 새로고침으로 반영 확인.

## Success metrics
- [x] 비개발자가 명령을 외우지 않고 **설정 상태 파악 + 정확한 명령 획득**을 웹에서 완료. ✔ 브라우저 dogfood(설정 페이지 배지·복사 렌더 확인)
- [x] **새 쓰기(POST/PUT/DELETE) 엔드포인트 0** — 공격면 불변. ✔ 라우트 열거 테스트(GET only)
- [x] 상태 배지가 **정확히 ok/미설정/확인불가**를 판정. ✔ 단위 테스트 19종 + 실환경 dogfood 일치
- [x] 응답에 **시크릿 원문 0** — 존재 여부만 노출. ✔ 함수+HTTP 테스트 + dogfood(curl 검사)

## Non-goals
- 웹에서 `.env`·토큰·MCP 설정을 **쓰는 것**(선택지 B/C 배제 — 2026-07-07 결정, decision 노트).
- Claude 구독 인증(`claude setup-token`)의 웹 대체 — 대화형 브라우저 OAuth라 웹 불가.
- **Cursor/Cline 연결 상태 감지** — `.cursor/mcp.json`은 프로젝트별이라 위치 불확정(복사 안내만).
- 시크릿(Gemini 키 등)의 웹 폼 입력.

## Constraints
- **읽기 전용 불변**: UI 라우터는 GET만. 쓰기 엔드포인트 추가 금지(034 보안 경계 계승).
- **시크릿 보호**: 서버측 마스킹 유지. 상태는 값이 아니라 존재 여부로만 판정·노출.
- repo **밖** 사용자 config(예: `claude_desktop_config.json`)는 **읽기만**, 쓰기 없음.
- 기존 034 디자인 시스템(토큰·`.badge` 클래스) **재사용** — 새 토큰 최소화(외과적 변경).

## Stakeholders
단일 사용자(localmind를 설치한 개인 누구나 — 비개발자 포함).

## Risks
- **상태 오탐**: 실제 연결됐는데 "안됨"으로 표시되면 신뢰 저하 → 판정 불가 시 '확인 불가'로 안전 저하.
- **config 경로 편차**: repo 밖 config 위치가 OS/앱 버전에 따라 다를 수 있음 → Live-Verify + graceful degrade.
- **클립보드 API 미동작 환경**: 폴백(텍스트 선택/수동 복사) 필요.
