# localmind, 비유로 이해하기

> 처음 보면 "게이트웨이? 임베딩? RAG? MCP?" 용어가 한꺼번에 쏟아집니다.
> 이 문서는 **딱 하나의 비유**로 전부 풀어줍니다. 5분이면 큰 그림이 잡힙니다.
> 정확한 기술 설명·설치는 [README](../README.md)로 가세요.

---

## 한 문장으로

**localmind는 "내 컴퓨터 안에 차린 1인 비서실"입니다.**

이미 월정액으로 고용한 똑똑한 직원(`claude`·`codex` CLI 구독)이 있는데,
지금은 그 직원이 **터미널 안에만** 갇혀 있습니다.
localmind는 그 직원을 위해 **안내데스크 · 수첩 · 서재 · 표준 호출 벨**을 차려줘서,
어떤 앱·어떤 코드에서도 불러 쓸 수 있는 **제대로 된 비서실**로 만들어 줍니다.

그리고 이 비서실은 **내 건물 안에만** 있습니다. 본사도, 공유 계정도 없습니다.

---

## 큰 그림 — 비서실 한 장면

```
        손님(내 코드 / Cursor / Claude Desktop)
                      │
              ┌───────▼────────┐
              │   안내데스크     │  ← 게이트웨이
              │  (통역 + 접수)   │
              └───────┬────────┘
        ┌─────────────┼──────────────┐
        ▼             ▼              ▼
   고용한 직원      수첩           서재 사서
  (claude/codex)   (메모리)      (second-brain)
        │             │              │
        └──── 전부 내 건물 안, 내 계정으로 ────┘
```

손님이 어떤 언어로 와도 **안내데스크**가 접수해서 알맞은 곳으로 넘기고,
직원은 일하면서 **수첩**에 핵심을 적고, 모르는 건 **서재**를 뒤져 답합니다.
이 모든 게 **내 컴퓨터(내 건물) 안**에서 끝납니다.

## 부품을 하나씩 — 무엇에 비유되나

### 🎙️ 게이트웨이 = 안내데스크 겸 통역사
손님(앱·코드)은 보통 **OpenAI나 Anthropic 말투**로 주문합니다.
그런데 내가 고용한 직원은 **터미널 CLI 말투**만 알아듣습니다.
게이트웨이는 그 사이에서 **통역**합니다.

> 그래서 기존 코드는 **주소(`base_url`) 한 줄만** localmind로 바꾸면 끝.
> 마치 해외 콘센트에 **변환 플러그**만 끼우는 것과 같습니다 — 기기는 그대로.

### 📐 임베딩 = 문장에 '의미 좌표'를 붙이는 색인 작업
도서관에서 책마다 **분류번호 스티커**를 붙여 비슷한 책을 같은 칸에 꽂듯,
임베딩은 문장마다 **숫자 좌표**를 붙여 "뜻이 비슷한 것끼리 가깝게" 만듭니다.
나중에 "이거랑 비슷한 내용 찾아줘"가 가능해지는 **밑작업**입니다.
(검색·메모리·서재가 전부 이 색인 위에서 돌아갑니다.)

### 📒 메모리(mem0) = 비서의 수첩
비서가 대화를 **통째로 녹음**하지는 않습니다. **핵심 사실만** 수첩에 적습니다.
- "사장님은 매운 거 못 드심"
- "결제 모듈은 9월에 교체하기로 결정"

그리고 사실이 바뀌면 **수첩을 고쳐 씁니다**(진화하는 기억). 다음에 `recall`하면
"그때 그렇게 정했죠"라고 **알아서 떠올려** 줍니다.

### 📚 second-brain = 내 서재를 다 읽은 전담 사서
내 노트 폴더(`.md` 파일들)를 가리키기만 하면, 사서가 그걸 **전부 읽어 색인**해 둡니다.
질문하면 통째로 외우는 게 아니라 **관련 페이지만 펼쳐서** 근거로 답합니다.
이게 흔히 말하는 **RAG**(읽고 → 찾아서 → 답하기)입니다.

> 수첩(메모리)과 서재(second-brain)의 차이:
> **수첩**은 일하며 자동으로 쌓이는 *짧은 사실*, **서재**는 내가 미리 써 둔 *긴 문서*.

### 🔌 MCP = 어디서든 비서를 부르는 '표준 호출 벨'
Cursor, Claude Desktop, Cline… 앱마다 비서를 부르는 방법이 다르면 피곤합니다.
MCP는 **공통 호출 규격**입니다. 한 번 등록해 두면 그 앱들이
`ask`(질문) · `remember/recall`(수첩) · `search_notes/ask_brain`(서재)를
**자기 도구처럼** 손에 쥡니다.

> 콘센트 규격(220V)을 통일해 두면 어떤 가전이든 꽂아 쓰는 것과 같습니다.

## 그래서 뭐가 좋은가

### 💸 "메터드 API 0원" = 미터기가 안 돈다
보통 클라우드 AI API는 **택시처럼 쓴 만큼 미터기**가 올라갑니다.
localmind는 **이미 낸 월정액 구독(claude·codex)** 안에서 다 해결합니다.
호출할 때마다 따로 빠져나가는 **종량 요금이 없습니다**.

### 🏠 "독립 실행" = 본사 없는 자영업 비서실
체인점이 아니라 **자영업**입니다. 각 컴퓨터는 자기 비서실을 **혼자** 운영합니다.
- 추론은 **그 머신 자신의 로그인** 계정으로 (공유 계정 ❌)
- 데이터(수첩·서재)는 **그 머신 로컬**에만
- 중앙 서버가 죽어도 **나는 멀쩡** (단일 장애점 없음)

여러 명/여러 서버는 **각자 독립 비서실**로 돌리고,
필요할 때만 원격으로 **"옆 건물 비서실에 전화"**(원격 MCP, URL+토큰)를 겁니다.
그래도 일은 **각자 자기 계정**으로 합니다.

## 비유 ↔ 실제 한눈에 보기

| 비유 | 실제 정체 | 한 일 |
|---|---|---|
| 안내데스크·통역사 | **게이트웨이** | OpenAI/Anthropic 요청 ↔ CLI 변환 |
| 고용한 직원 | **claude / codex CLI** | 실제 답변 생성(이미 구독한 것) |
| 분류번호 스티커 | **임베딩(bge-m3)** | 문장 → 의미 좌표 |
| 비서의 수첩 | **메모리(mem0)** | 핵심 사실 저장·갱신·회상 |
| 서재 사서 | **second-brain(RAG)** | 내 `.md` 노트로 근거 답변 |
| 표준 호출 벨 | **MCP 도구** | 외부 앱에서 위 기능을 도구로 |
| 변환 플러그 | **`base_url` 교체** | 기존 코드 그대로 연결 |
| 자영업 비서실 | **독립 실행 원칙** | 자기 계정·자기 데이터·자립 |
