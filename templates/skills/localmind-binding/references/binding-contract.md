# 바인딩 소비 규약 (정본)

다른 워크플로 스킬이 페르소나·모델이 필요한 시점에 이 설치의 바인딩을 읽을 때 지켜야 할
규약이다. 저장은 `/localmind-binding` 온보딩 스킬이 맡고, 이 문서는 **읽는 쪽**의 계약이다.

## 파일 위치

`<데이터 폴더>/_bindings/<runtime-id>.json` — 설치(런타임)마다 파일 하나. `_` 접두는
"기본 로컬·미커밋" 컨벤션이며, 이 폴더는 백업·동기화 파이프라인에서 제외된다.

## 스키마

```json
{
  "schemaVersion": 1,
  "runtime": "<runtime-id>",
  "updatedAt": "<YYYY-MM-DD>",
  "tiers": {
    "critical-reasoning": { "model": "<모델 식별자>" },
    "standard": { "model": "<모델 식별자>" },
    "economy": { "model": "<모델 식별자>" }
  },
  "roles": {
    "<역할명>": { "persona": "<페르소나 name>", "tier": "critical-reasoning" }
  }
}
```

- `tiers`는 세 등급 중 일부만 있어도 유효하다(부분 설정). 세 등급 밖의 알 수 없는 키는 오류다.
- `roles`의 키(역할명)는 자유롭다 — 이 계약은 역할 집합을 고정하지 않는다. 각 값은
  `persona`(레지스트리에 존재해야 함, 저장 시점 검증)와 `tier`(세 등급 중 하나)를 가진다.
- 역할의 실행 모델은 `roles.<역할>.tier`를 경유해 `tiers[tier].model`로 해석한다(직접 지정
  필드는 없다).

## 식별자 도출과 정확 일치

읽는 쪽도 같은 규칙으로 자기 식별자(runtime-id)를 스스로 도출한 뒤, **그 식별자와 정확히
일치하는 파일만** 읽는다. 일치하는 파일이 없으면 다른 설치의 바인딩 파일을 대신 읽지 않고,
아래 "부재 시 정책"을 그대로 적용한다 — 필요하면 `_bindings/` 안의 기존 파일 목록을 보여줘
사용자가 표기 차이를 스스로 바로잡게 한다.

## 부재 시 정책

필요한 항목(등급의 모델 또는 역할의 페르소나)이 없으면, **어떤 side-effect도 일으키기 전에**
평이한 한국어로 온보딩 방법을 안내하고 **기본적으로 진행하지 않는다**. 등급·역할 일부만 있는
바인딩에서도 이 규칙은 미설정 항목에만 적용한다 — 설정된 항목까지 무효로 취급하지 않는다.

예외로, 사용자가 그 안내에 대해 **그 자리에서 명시적으로** "이번만 바인딩 없이 진행"을
선택하면 진행할 수 있다. 이 경우 아무것도 저장하지 않고, 결과 보고에 바인딩 미설정 상태로
진행했음을 임시로 명시한다.

## 페르소나 대행(비독립 fallback)

역할 → 페르소나로 위임하는 능력 자체가 없는 설치에서는, 그 역할의 페르소나가 바인딩에
있더라도 위임할 수 없다. 이 경우 **현재 세션이 그 역할의 체크리스트를 직접 수행**하고,
결과 보고에 그것이 **비독립(fallback)**임을 명시한다. 페르소나 위임 불가·바인딩 미설정
어느 쪽도 워크플로를 중단시키지 않는다.

## 모델 우선순위 — 페르소나 정의가 바인딩 tiers보다 우선한다

역할을 별도 프로세스로 격리 위임하는 것이 실제로 가능한 설치에서는, 그 위임된 프로세스가
쓰는 실행 모델의 정본은 **페르소나 정의 자체가 담은 model 값**이다 — 바인딩의 tiers가
아니다. 바인딩의 tiers는 다음 두 경우에만 쓴다 — ① 페르소나 위임이 불가능해 현재 세션이
대행(비독립 fallback)하는 경우, ② 워크플로가 격리 위임과 무관하게 그 자리에서 직접 모델을
골라야 하는 경우. **정본은 하나다** — 위임이 실제로 일어나면 페르소나 정의의 model 값이
바인딩 tiers보다 우선하며, 워크플로는 이 우선순위를 뒤집지 않는다.

## 모델 가용성

바인딩에 저장된 모델 식별자는 실제로 쓸 수 있는지(가용성) 검증되지 않은 값일 수 있다.
저장 시점에도, 읽는 시점에도 이 계약은 가용성을 프로빙하지 않는다 — 읽는 쪽은 이 값을
검증되지 않은 사용자 확정값으로 다룬다.
