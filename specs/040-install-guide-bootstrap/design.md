# Design: 설치 전 부트스트랩 가이드 (디자인 사전 정의)

> 디자인 게이트: 이 문서 확인 전 UI 구현 착수 금지(AGENTS.md). **비개발자용**이라 이 문서는
> 시각·토큰뿐 아니라 **가이드 콘텐츠(무엇을·왜·어떻게·막히면)** 까지 정본으로 못 박는다.
> 토큰 정본은 034 `public/ui/style.css` — 서버가 서빙해 **재사용**한다.

## 1. 디자인 시스템 패턴
- **패턴**: **세로 단계형 가이드**(진행 요약 → 섹션별 카드 → 단계 카드). 각 단계 = 번호·제목·설명·
  명령+복사·예상 결과·▸접이식 "막히면". 준비물은 상태 배지로 라이브 표시.
- **근거**: 비개발자는 "위에서 아래로 하나씩"이 가장 안전. 진행 요약으로 **상태 가시성**, 접이식
  문제 해결로 **정보 과부하 방지**. 034 웹과 시각 일관성(같은 토큰).
- **기각**: *가로 스텝퍼/위저드* — 모바일·초심자에 취약, 되돌아보기 어려움. *터미널 로그만* — 원래 문제.

## 3. 컴포넌트 정의

### 3.1 준비물 점검 행 + 상태 배지 (`.badge` 재사용)
- **목적**: 필수/선택 준비물이 지금 됐는지 라이브 표시.
- **배지 상태**(높이 20px, `2px 8px`, radius 999px, `--font-caption`):
  | 상태 | 클래스 | 문구 | 의미 |
  |---|---|---|---|
  | ok | `.badge.ok` | `됨` | 점검 통과 |
  | missing(필수) | `.badge.warn` | `안됨` | 미충족 — 아래 설치 안내 따르기 |
  | missing(선택) | `.badge.idle` | `선택 · 없음` | 없어도 진행 가능(ollama 등) |
  | unknown | `.badge.idle` | `확인 불가` | 점검 실패 → 수동 확인 |
  | loading | `.badge.idle` | `확인 중…` | `/api/checks` 응답 전 |
- **전이**: 진입/새로고침 → 전부 `확인 중` → 응답 도착 시 확정. 실패 → `확인 불가`.

### 3.2 명령 복사 버튼 `.copy-btn` (신규)
- **치수**: 높이 `28px`, 패딩 `0 var(--space-sm)`, radius `6px`, `1px solid var(--color-border)`,
  배경 `--color-surface`, `--font-caption`, `cursor:pointer`. 명령 chip과 gap `--space-sm`.
- **상태·전이**:
  | 상태 | 표시 | 전이 |
  |---|---|---|
  | idle | "복사" | 기본 |
  | hover/focus | 테두리 `--color-primary` | 포커스 |
  | copied | 배경 `--color-ok`·글자 `--color-on-state`·"복사됨 ✓" | 클릭 성공 → 1.5s 후 idle |
  | 폴백 | 버튼 숨김·chip이 `user-select:text`·"드래그해 복사" | `navigator.clipboard` 미가용 |
- localhost(127.0.0.1)는 보안 컨텍스트라 clipboard 가용 — 폴백은 예외 대비.

### 3.3 명령 chip `.cmd`
- `--font-mono`·배경 `--color-bg`·패딩 `--space-xs var(--space-sm)`·radius 6px. 길면 `overflow-x:auto`
  (자체 가로 스크롤, 페이지 본문은 스크롤 안 함).

### 3.4 단계 카드 `.step-card` / 준비물 카드
- 기존 `.card`(surface·border·radius-card·패딩 md) 재사용. 단계마다 **원형 번호 뱃지**
  (`--step-number-size`, 배경 `--color-primary`·글자 `--color-on-primary`) + 제목(`--font-title`).
- 내부 구조: `무엇/왜`(dim) → `명령 chip + 복사`(있으면) → `예상 결과`(dim) → `▸ 막히면`(접이식).

### 3.5 접이식 "막히면 / 문제 해결" `.collapsible`
- 기본 접힘(▸), 클릭 시 펼침(▾). `<details><summary>` 네이티브 요소 사용(무의존·접근성 무료).
- 내용: 증상 → 원인 한 줄 → 해결 명령(복사) 또는 링크(docs/troubleshooting.md).

### 3.6 진행 요약 바
- 상단 고정 아님(스크롤). "준비물 N/4 됨" + 각 항목 미니 점(●ok/○idle). 클릭 시 해당 준비물로 스크롤(선택).

## 4. 콘텐츠 정본 (비개발자 눈높이 — 실제 문구 기준)

> 구현은 이 콘텐츠를 **정본**으로 삼는다(임의 축약 금지). Live-Verify 대상(설치 링크·명령)은
> 구현 시 최신 확인.

**준비물(필수)**
- **Docker Desktop** — 무엇: "여러 프로그램을 서로 간섭 없이 돌리는 '컨테이너' 도구". 왜: "localmind
  두뇌가 이 위에서 돕니다". 설치: 공식 다운로드 링크(Mac/Windows). 막히면: "설치 후 **실행**해서
  상단 메뉴바에 고래 아이콘이 뜨고 'Running'이 될 때까지 기다리세요".
- **Node.js 20+** — 무엇: "앱과 localmind를 잇는 도구". 왜: "MCP 연결에 필요". 설치: nodejs.org LTS.
  막히면: "설치 후 터미널을 새로 열고 `node -v`가 v20 이상인지 확인".
- **make / git** — 보통 기본 설치(Mac: Xcode Command Line Tools, `xcode-select --install`). 막히면 링크.

**준비물(선택)**
- **ollama** — 무엇: "임베딩(의미 검색)을 더 빠르게(Mac Metal)". 왜: "없어도 되지만 있으면 빠름".
  설치 전엔 없음이 정상 → `선택 · 없음`.

**설치 단계**
1. **터미널 열기** — Mac: `⌘Space` → "터미널". Windows: 시작 → "PowerShell". (스크린샷/아이콘 설명)
2. **받기** — `git clone <repo>` → `cd localmind`. 예상: 폴더 생성 + 이동. (이미 clone했으면 건너뛰기)
3. **설치 실행** — `make setup`. 무엇: 준비물 점검→설치→연결을 한국어로 안내. 시간: 수 분. 질문이
   나오면 대개 Enter 또는 `y`. 막히면 → 3.문제 해결.

**다음 단계** — 설치 완료 후: 첫 사용 튜토리얼(docs/tutorial.md) · Claude Desktop 연결 `make mcp-desktop`.

**문제 해결(접이식 최소 3종)**
- "docker compose 에러/Gordon" → Docker Desktop이 실행 중인지 확인 후 다시 `make setup`.
- "make: command not found" → Mac `xcode-select --install`, 그 뒤 다시.
- "포트가 이미 쓰인대요" → docs/troubleshooting.md 링크.

## 5. 반응형
- 본문 `--guide-max-width: 760px`, 가운데 정렬. 모바일(<480px): 카드 패딩 `--space-sm`, 번호 뱃지·
  제목 한 줄 유지, 명령 chip은 `overflow-x:auto`(페이지 가로 스크롤 금지).
- 진행 요약 점들은 좁으면 줄바꿈(`flex-wrap`).
- 다크 모드: 토큰(라이트/다크 자동)만 사용 → 자동 대응.

## 6. 접근성
- 배지는 색+문구 이중(색맹 대비). 복사 `<button>`+`aria-label`, "복사됨"은 `aria-live="polite"`.
- 접이식은 네이티브 `<details>`(키보드·스크린리더 기본 지원).
- 본문 대비: `--color-text` on `--color-bg`(WCAG AA 목표). 링크는 밑줄+색.

## 7. 에이전트 실행 프롬프트

### worker(구현)에게
```
specs/040-install-guide-bootstrap의 spec.md·design.md 기준으로 구현하라.
- scripts/bootstrap-guide.mjs는 Node 내장 모듈만 쓴다(http·fs·child_process·os·path).
  외부 패키지·tsx·dist·node_modules import 금지(무의존 불변 — AC-1).
- 점검은 순수 classifyCheck + spawn 어댑터로 분리하고, 실패/타임아웃은 unknown으로 흡수(AC-5,7).
- HTTP는 GET만: '/'(HTML)·'/api/checks'(JSON)·'/style.css'(034 재사용). 명령 실행 엔드포인트 금지(AC-7).
- 색/배지는 034 토큰·.badge 재사용. 신규 CSS는 guide.css(.copy-btn·.step-card·번호뱃지·collapsible)만.
- 복사 버튼: clipboard 성공 시 copied(1.5s)→idle, 미가용 시 선택 폴백(AC-6). 접이식은 <details>.
- 콘텐츠는 design.md 4장을 정본으로(비개발자 눈높이 — 무엇/왜/예상/막히면, '터미널 여는 법' 포함, AC-11).
- make guide는 node 존재 확인 후 실행, 없으면 안내+링크 후 비크래시 종료(AC-8). 127.0.0.1만(AC-10).
```

### ux-reviewer(점검)에게
```
구현된 가이드를 design.md와 대조하라.
- 콘텐츠 충실도: 각 단계가 무엇/왜/명령/예상/막히면을 갖추고, 터미널 여는 법 등 초심자 전제가 있는가(AC-11).
- 상태 가시성: 준비물 배지가 ok/missing(필수·선택)/unknown/확인 중을 색+문구로 구분하는가.
- 토큰 준수: 034 토큰 재사용·하드코딩 색 없음. 복사 버튼/접이식 상태 전이가 정의대로인가.
- 접근성: 색+문구 이중, button+aria-label+aria-live, <details> 사용, 대비.
- 반응형: <480px에서 명령이 페이지 가로 스크롤을 만들지 않는가.
- 경계: 브라우저에서 명령을 '실행'하는 UI가 없는가(복사만).
최종 품질 판정은 크리틱(다운시프트 금지).
```

## 8. tokens.json (해당 없음)
