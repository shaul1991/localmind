# Goal: Persona Agent Registry — 페르소나 에이전트 정의의 정본 관리·배포·싱크

## Background — 배경

localmind 사용자는 여러 AI 코딩 도구(Claude Code, Codex CLI)를 병행하고, 여러 기기를
오가며 작업한다. 작업 품질을 높이기 위해 역할별 전문 페르소나 에이전트(예: 도메인 전문가,
QA 리드, 적대적 리뷰어)를 정의해 위임하는 방식이 유효함이 경험적으로 확인됐지만, 그 정의는
각 도구의 전용 폴더에 도구별 형식으로 흩어져 있다.

localmind는 이미 "파일이 정본, 파생물은 재생성, git이 백업·싱크"라는 원칙으로 노트와
메모리를 관리한다. 페르소나 에이전트 정의도 같은 원칙 위에서 관리할 수 있는 자산이다.

이 작업은 페르소나 위임 체계(레지스트리 → 런타임 위임 → 오케스트레이션) 3단계 중
첫 단계로, 이후 단계(localmind 런타임의 페르소나 위임, 작업 워크플로우 오케스트레이션)의
토대가 된다.

## Problem — 문제

- 에이전트 정의가 특정 도구 전용 위치에만 존재해 **버전 관리·백업·기기 간 싱크가 안 된다.**
  기기를 새로 세팅하면 에이전트 체계가 통째로 사라진다.
- 같은 페르소나를 Claude Code와 Codex에서 함께 쓰고 싶어도 **도구마다 정의 형식이 달라**
  이중 관리가 필요하고, 실제로는 한쪽(Claude Code)에만 존재한다.
- 페르소나별로 **어떤 AI 모델·추론 강도를 쓸지**가 정의의 일부인데, 이를 한 곳에서
  선언하고 도구별 설정으로 일관되게 반영할 방법이 없다.
- localmind에는 에이전트라는 개념 자체가 없어, 이후 단계(런타임 위임)를 시작할 기반이 없다.

## Objective — 목표

페르소나 에이전트 정의를 localmind가 관리하는 **단일 정본(레지스트리)** 으로 두고,
거기서 각 도구(Claude Code, Codex)가 실제로 사용하는 형식으로 **배포(내보내기)** 할 수
있게 한다. 정의에는 페르소나의 역할·지침과 함께 **대상 도구별 모델·추론 강도**가
포함되며, 기존 백업·복구 흐름에 얹혀 **기기 간 싱크와 복원**이 된다.

## Expected outcome — 기대 결과

- 페르소나를 한 번 정의하면 Claude Code와 Codex 양쪽에서 같은 페르소나를 쓸 수 있다.
- 페르소나 정의의 변경 이력이 git으로 추적되고, 백업 저장소를 통해 기기 간 동기화된다.
- 새 기기에서 복구하면 노트·메모리와 함께 페르소나 체계도 복원된다.
- 이후 스펙(런타임 위임·오케스트레이션)이 이 레지스트리를 참조해 페르소나를 해석할 수 있다.

## Success metrics — 성공 지표

- 하나의 정본 정의에서 한 명령으로 Claude Code용·Codex용 산출물이 모두 생성된다.
- 새 기기에서 복구 절차 + 배포 한 번으로 기존 기기와 동일한 페르소나 체계가 재현된다.
- 배포를 반복 실행해도 결과가 같다(멱등) — 수동으로 만든 무관한 파일은 건드리지 않는다.
- 페르소나별 모델·추론 강도 지정이 각 도구의 실제 동작(모델 선택)에 반영된다.

## Non-goals — 비목표

- **페르소나 구성(어떤 페르소나를 둘 것인가) 확정** — 별도 설계 인터뷰로 진행한다.
  이 작업은 그릇(스키마·배포·싱크)만 만들고, 검증용 최소 샘플만 포함한다.
- **localmind 런타임의 페르소나 위임** (ask/ask_brain의 페르소나 파라미터) — 다음 스펙.
- **오케스트레이션** (SDD 단계 팬아웃, Codex exec 위임 스킬·피드백 루프) — 다음 스펙.
- **역방향 임포트** (도구 전용 폴더의 기존 정의 → 레지스트리 자동 변환) — 페르소나 재구성
  인터뷰 결과에 따라 필요 시 별도 진행.
- **양방향 동기화** — 배포는 정본→파생 단방향이다. 파생 파일의 수동 편집은 다음 배포에서
  덮어써질 수 있다(정본에서 고친다).

## Constraints — 제약

- 정본은 사람이 읽고 편집할 수 있는 평문 파일이어야 한다(localmind 원칙: 파일이 정본).
- 기존 백업·복구 흐름을 변경 없이 재사용한다 — 새 동기화 채널을 만들지 않는다.
- 대상 도구가 설치되지 않은 기기에서도 나머지 기능은 정상 동작해야 한다(graceful).
- 비개발자 사용자가 이해할 수 있는 평이한 한국어 메시지·문서를 제공한다.
- 배포는 명시적 실행이 기본이다 — 사용자가 모르는 사이 도구 설정이 바뀌지 않는다.

## Stakeholders — 이해관계자

- 단일 사용자(설치한 개인 누구나 — 비개발자 포함). 여러 AI 도구·여러 기기를 쓸수록
  효용이 커진다.

## Risks — 리스크

- **도구 형식 드리프트**: Claude Code·Codex의 에이전트/설정 형식이 버전업으로 바뀌면
  배포 산출물이 무효화될 수 있다 → 어댑터를 도구별로 분리해 국소화한다.
- **파생 파일 충돌**: 사용자가 도구 폴더에 직접 만든 파일을 배포가 덮어쓰면 데이터
  유실이다 → localmind가 만든 파일만 관리 대상으로 식별해 그 외에는 손대지 않는다.
- **민감정보 유출**: 페르소나 지침에 민감정보를 넣으면 백업 저장소에 그대로 커밋된다 →
  문서로 경고한다(기존 백업 기능과 동일한 리스크·동일한 대응).
