# Goal: 웹 설치 위저드 — 비개발자가 브라우저에서 설치를 끝낸다

## Background — 배경

localmind는 비개발자까지 사용자로 삼는 오픈소스 개인 second-brain이다(AGENTS.md). 설치는 이미
`make setup`(5단계 대화형 가이드)로 한 명령이며, 준비물 감지·`.env`/키 자동 생성·`.env` 자동 기록이
갖춰져 있다. 또한 **의존성 없는 웹 부트스트랩 서버 `make guide`(scripts/bootstrap-guide.mjs, Node
내장 모듈만)**가 이미 존재해 설치 전에 브라우저에서 준비물 배지와 복사용 명령을 보여준다.

그러나 `make guide`는 **수동 안내판**이다 — 명령을 보여줄 뿐 대신 실행하지 않고, 진행 상황을
스트리밍하지 않으며, 토큰 발급·MCP 등록을 자동화하지 않는다.

## Problem — 문제

비개발자가 반복적으로 막히는 지점은 "명령 수"가 아니라(이미 한 명령) 다음의 암묵/불투명/조작
마찰이다(specs/046 사전 진단):

1. **암묵 단계**: Docker Desktop 설치 후 "앱을 한 번 실행"해야 하는 단계를 모른다.
2. **토큰 조작**: claude 구독 토큰·Gemini 키를 어디서 발급받고 어떻게 붙여넣는지 모른다(파일 편집 미경험).
3. **무음 대기**: 첫 `docker compose` 기동에서 이미지·임베딩 모델(~1.2GB) 내려받기로 5~15분 걸리는데
   진행 표시가 없어 멈춘 건지 알 수 없다.
4. **MCP 확인**: "Claude Code를 완전히 종료했다 다시 켜"가 무엇이고, 도구가 어디에 나타나는지 불명확하다.
5. **용어**: Docker·MCP·bge-m3·토큰 같은 용어가 개념화되지 않아 각 단계의 의미를 모른다.

## Objective — 목표

기존 `make guide`(부트스트랩 서버)를 **능동 웹 설치 위저드로 승격**한다 — 브라우저에서 안전한
화이트리스트 설치 단계를 **대신 실행**하고(진행 스트리밍), 토큰 발급·MCP 등록을 안내·자동화해
비개발자가 **터미널 조작을 최소화한 채 브라우저에서 설치를 완료**하게 한다. 이번 범위는 **macOS 우선**.

## Expected outcome — 기대 결과

- 사용자가 한 명령(`make guide`)으로 브라우저 위저드를 연다.
- 위저드가 준비물을 실시간 점검(배지)하고, 빠진 것은 "받기/열기" 안내를 제시한다.
- 사용자 확인을 거쳐 위저드가 스택 기동(docker) 등 **화이트리스트 단계를 대신 실행**하고
  **진행 상황을 스트리밍**(예: 모델 내려받기 진행)한다.
- 토큰 발급을 브라우저 흐름으로 안내하고 결과를 `.env`에 **서버측에서** 기록한다(값은 화면 노출·전송 최소화).
- MCP 등록을 수행하고 "도구가 등록됐는지"를 자동 폴링으로 확인해 표시한다.
- 완료 시 "이제 이렇게 쓰세요"로 넘긴다. 기존 CLI(`make setup`)는 그대로 병존한다.

## Non-goals — 비목표

- **멀티 사용자·사용자별 격리**(단일 사용자 전제 유지).
- **Windows/Linux 지원**(이번 범위 제외 — macOS 우선, 후속 스펙).
- **원클릭 무확인 풀오토 설치**(각 단계 사용자 확인 유지 — 능동이되 확인 게이트).
- **기존 CLI(`make setup`) 대체**(위저드는 병존·보완이며 CLI를 제거하지 않는다).
- **웹에서 임의 `.env` 편집·임의 셸 실행**(설정은 안내된 값만, 실행은 화이트리스트만).
- **npx/curl 한 줄 신설 등 새 배포 진입로**(이번엔 기존 `make guide`(clone+node 1회) 확장 — 별도 배포 인프라는 후속).

## Constraints — 제약

- **보안 우선(로컬 특권 에이전트)**: 위저드는 docker 실행·`.env` 기록·MCP 등록을 하는 특권 서버다.
  바인딩은 `127.0.0.1` 전용, Host 헤더 검증(DNS rebinding 차단), 실행은 **고정 화이트리스트**로만.
  비밀값은 서버측에만 두고 클라이언트로 보내지 않는다.
- **부트스트랩 무의존 유지**: 설치 전에 떠야 하므로 진입 서버는 `bootstrap-guide.mjs`처럼 **Node 내장
  모듈만**으로 동작한다(node_modules 전제 금지). 능동 단계는 기존 스크립트를 호출한다.
- **기존 자산 재사용**: `setup.sh`·`up.sh`·`mcp-install.sh`·`doctor.sh`의 로직을 재구현하지 않고
  위저드가 그 스크립트를 실행 원시로 호출한다.
- **비개발자 언어**: 화면 문구·상태·에러는 평이한 한국어. 용어는 짧은 설명/비유 병기. 절대경로 플레이스홀더.
- **비파괴**: 실패해도 기존 설정·데이터를 손상하지 않고 재시도 가능(기존 스크립트의 비파괴 관례 계승).

## Stakeholders — 이해관계자

- 단일 사용자: localmind를 설치하는 개인 누구나(특히 **비개발자**).
- 유지보수자(설치 지원 부담을 줄이려는).

## Risks — 리스크

| Risk | Impact | Mitigation |
|---|---|---|
| 특권 실행 노출(화이트리스트 우회) | 임의 명령 실행 벡터 | 127.0.0.1 전용·Host 검증·**고정 화이트리스트**(임의 문자열 실행 금지)·확인 게이트 |
| 토큰/비밀 유출 | 개인 자격 노출 | 서버측 기록만·클라이언트 미전송·화면 마스킹(기존 :8788 관례 계승) |
| 첫 docker 대기(5~15분) UX | 사용자가 멈춘 걸로 오인·이탈 | 진행 스트리밍·진행률·"한 번만 오래" 안내 |
| 터미널 진입 자체는 남음 | 완전 무터미널 아님 | 진입을 `make guide` 한 줄로 최소화(무터미널화는 후속 npx 스펙) |
| 능동성 범위 확대(풀오토로 드리프트) | 복잡·위험 증가 | 각 단계 확인 게이트를 goal 제약으로 고정 |
| 기존 CLI/스크립트와 이중 유지 | 드리프트 | 위저드가 스크립트를 재구현하지 않고 호출(단일 로직원) |
