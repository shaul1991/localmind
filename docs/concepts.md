# 비유로 이해하기 — 5분 큰 그림

> 처음 보면 "임베딩? RAG? MCP?" 용어가 한꺼번에 쏟아집니다.
> 이 문서는 **비유 하나**로 전체 그림을 잡아줍니다. (설치·사용법은 [README](../README.md) 참고)

---

## 한 문장으로

**localmind는 "내 컴퓨터 안의 개인 서재 + 전담 사서"입니다.**

AI(Claude·Cursor…)와 일하며 생긴 결정·지식·맥락이 **서재의 문서(.md 파일)**로 쌓이고,
어떤 AI 앱이든 그 서재를 **사서에게 물어** 꺼내 씁니다. 서재는 **내 건물 안에만** 있습니다 —
본사도, 공유 계정도, 클라우드 저장도 없습니다.

---

## 등장인물 셋

### 📚 서재 = 내 노트 폴더 (`.md` 파일들 — 정본)
모든 기억의 정본은 **평범한 마크다운 파일**입니다. 폴더를 열면 언제든 직접 읽고·고치고·지울 수
있습니다. AI가 대화 중 "기록해둬" 한마디에 문서를 꽂아 넣습니다(`capture_note`).
가장 값진 문서는 **"왜 그렇게 결정했는가"**의 기록입니다.

### 📐 색인 = 문장에 '의미 좌표'를 붙이는 스티커 작업 (임베딩)
도서관에서 책마다 **분류번호 스티커**를 붙여 비슷한 책을 같은 칸에 꽂듯, 임베딩은 문장마다
**숫자 좌표**를 붙여 "뜻이 비슷한 것끼리 가깝게" 만듭니다. 그래서 단어가 정확히 같지 않아도
"그거 뭐였지?"로 찾아집니다(`search_notes`). 스티커 작업은 같은 컴퓨터의 작은 엔진
(Ollama의 bge-m3)이 합니다 — 외부 전송 없음.

### 🔌 MCP = 어디서든 사서를 부르는 '표준 호출 벨'
Cursor, Claude Desktop, Cline… 앱마다 사서를 부르는 방법이 다르면 피곤합니다.
MCP는 **공통 호출 규격**입니다. 한 번 등록해 두면 그 앱들이
`capture_note`(꽂기) · `search_notes`(찾기) · `whoami`(어느 서재인지)를 **자기 도구처럼** 손에 쥡니다.

> 콘센트 규격(220V)을 통일해 두면 어떤 가전이든 꽂아 쓰는 것과 같습니다.

## 그래서 뭐가 좋은가

### 🧠 "배경 재설명이 사라진다"
Cursor에서 정한 결정을 Claude Desktop이 압니다. 어제 기기에서 남긴 맥락을 오늘 기기가
갖고 시작합니다. **같은 서재를 보기 때문**입니다(같은 `NOTES_DIR`, git 백업/복구).

### 🏠 "전부 로컬" = 본사 없는 자영업 서재
- 데이터(노트·색인)는 **그 머신 로컬**에만
- 밖으로 나가는 호출은 임베딩뿐 — 그것도 기본은 **같은 컴퓨터의 Ollama**
- 중앙 서버가 없으니 **단일 장애점도 없음** — 백업은 내 git repo로

## 비유 ↔ 실제 한눈에 보기

| 비유 | 실제 정체 | 한 일 |
|---|---|---|
| 서재의 문서 | **`.md` 노트(정본)** | 결정·지식·맥락 보관 |
| 분류번호 스티커 | **임베딩(bge-m3)** | 문장 → 의미 좌표(색인) |
| 사서에게 묻기 | **의미 검색(search_notes)** | 관련 문서만 펼쳐 근거 제공 |
| 아침 브리핑 | **세션 브리핑(brief)** | 과거 결정(선택·이유·전제)을 들고 시작 |
| 표준 호출 벨 | **MCP 도구 4종** | 외부 앱에서 위 기능을 도구로(기록·검색·브리핑·식별) |
| 자영업 서재 | **local-first 원칙** | 내 파일·내 기기·자립 |
