# 사용법 — 매일 이렇게 씁니다

설치를 마쳤다면([README Quickstart](../README.md#quickstart)), 이제 **그냥 채팅창에서 말로** 씁니다.
Claude Desktop / Cursor에서 평소처럼 대화하되, 아래 네 가지를 자연어로 시키면 됩니다.

> 개념이 처음이면 👉 [비유로 이해하기](concepts.md) · 궁금증은 👉 [FAQ](faq.md)

---

## 한눈에 — 네 가지 흐름

```
① 쌓기(캡처)  →  ② 활용(묻기)  →  ③ 보기(열람)  →  ④ 정리(삭제)
   기억해둬        그거 뭐였지        다 보여줘         이거 지워줘
```

전부 **채팅창 안에서 말로** 끝납니다. 별도 화면·터미널 필요 없습니다.

---

## ① 쌓기 — 기억·노트 남기기

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 짧은 사실·결정 기억 | "**기억해둬** — 결제는 OAuth로 결정" | `remember` |
| 정리한 문서를 노트로 보관 | "이 내용 **노트로 저장**해줘" | `capture_note` |

- **기억(메모리)** = 일하며 쌓는 *짧은 사실* (mem0). "사장님은 매운 거 못 드심", "출시 11월로 연기".
- **노트(.md)** = 내가 쓴 *긴 문서*. `NOTES_DIR`(기본 `~/.localmind`)의 마크다운 파일이 정본.

## ② 활용 — 꺼내 쓰기

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 지난 결정·사실 떠올리기 | "결제 **뭐로 정했지**?" | `recall` |
| 내 노트 근거로 답 받기 | "내 노트 기준 우리 원칙이 뭐야?" | `ask_brain`(출처 인용) |
| 노트에서 관련 부분 찾기 | "백업 관련 노트 **찾아줘**" | `search_notes` |
| 다른 모델에 교차 질문 | "이거 codex한테도 물어봐줘" | `ask` |

→ **특별한 조작 없이** 평소처럼 물으면 localmind가 알아서 기억·노트를 뒤져 답합니다.

## ③ 보기 — 뭐가 쌓였나 둘러보기

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 기억 전체 목록 | "내 **기억 다 보여줘**" | `list_memories` |
| 노트 파일 목록 | "내 **노트 목록** 보여줘" | `list_notes` |
| 지금 어떤 두뇌인지 | "whoami" | `whoami` |

→ `recall`(의미검색)과 달리 `list_*`는 **전부 나열**해서 *무엇이 쌓였는지* 둘러볼 수 있습니다.

## ④ 정리 — 틀린 것 지우기

| 하고 싶은 것 | 이렇게 말하면 | 쓰는 도구 |
|---|---|---|
| 잘못된 기억 삭제 | "그 **기억 지워줘**" | `delete_memory` |
| 노트 파일 삭제 | "이 **노트 지워줘**" | `delete_note` |

→ 모델이 먼저 `list_*`로 목록·id를 확인한 뒤 지워줍니다. **삭제는 되돌릴 수 없으니** 보통 확인을 거칩니다.

---

## 두 종류 기억, 헷갈리면

| | 기억(메모리) | 노트(second-brain) |
|---|---|---|
| 무엇 | 짧은 사실 | 긴 문서(.md 파일) |
| 어떻게 쌓나 | "기억해둬" | "노트로 저장" / 폴더에 파일 |
| 어디 있나 | mem0 DB(로컬) | `NOTES_DIR`(기본 `~/.localmind`) 파일 |
| 보기 | `list_memories` | `list_notes` / 폴더 열기 |

---

## 도구를 바꿔 써도 — 같은 두뇌

Cursor에서 기억시킨 걸 Claude Desktop에서 꺼내 씁니다. 전제: 각 도구의 MCP 설정에서
**같은 `OPENMEMORY_USER`(+`NOTES_DIR`)** 를 쓰면 됩니다(= 같은 두뇌 공유). 단, *저장된* 것만
공유됩니다 — 방금 그 창의 저장 안 한 대화는 따라가지 않으니, 중요하면 "기억해둬"로 박아두세요.

## 여러 주제로 나누고 싶다면 (선택)

기본은 폴더 하나(`~/.localmind`)로 충분합니다. 주제별로 나누고 싶으면:
```
NOTES_DIR="work=/내/업무노트,life=/내/개인노트"
```
→ `list_notes`/`search_notes`/`ask_brain`에 `folder:"work"` 로 한정할 수 있습니다.

## 백업 · 새 기기 복구

- **백업**: `make backup` (메모리+노트를 내 private git repo로). 자동은 `make backup-cron`.
- **복구**: 새 컴퓨터에서 `make recover RESTORE_REPO=<내 백업 repo>` 한 줄.
- 자세히 👉 [README의 백업 섹션](../README.md#메모리-백업-git)

## 막히면

- 도구가 안 보임 → **Claude Desktop/Cursor 재시작**(MCP는 시작 시 로드)
- `remember`가 `User not found` → MCP 설정의 `OPENMEMORY_USER`가 스택이 시드한 값과 일치해야 함(기본 `localmind`)
- 더 많은 증상 👉 [README Troubleshooting](../README.md#문제-해결-troubleshooting)
