# goal — local-first 기본 재확정: repo 백업 + localhost 두뇌 (원격은 옵션)

## Background

- **사용자 제안(2026-07-21)**: "localmind를 다시 순수한 repo 기반 백업과 localhost 기반으로."
  같은 날 실측이 배경: 중앙 홈서버 3주 부패(릴리스 8개 미배포)·품질 텔레메트리 단절(로그는
  서버, 리포트는 로컬 빈 로그)·Claude Desktop 연결 불가(http+Bearer)·Tailscale 의존.
- **방향 결정 B(확정)**: local-first 기본 + 홈서버 원격은 씬 클라이언트용 옵션(은퇴 아님).
  M5 복원은 구현의 도그푸드로, Desktop 등록 복원 포함. (결정 노트 2026-07-21T01-12 적재.)
- **사실 지도 재프레임(핵심)**: 조사 결과 **로컬 모드는 제거된 적이 없다** — specs/045는
  stdio 기본에 http 전송을 *추가*한 것(F-2·F-4·F-5), README·usage 설치 서사도 이미 로컬
  기준(F-21~24), backup/restore/recover 체인 완비(F-15·F-16). 남은 실체는 (a) Ollama 직결의
  일급 문서화(.env.example에 EMBEDDINGS_URL 미기재 — F-20) **+ 설치·재색인 스크립트의 옵션
  패스스루 배선**(critic 실측: 현행은 죽은 노브 — 문서화된 노브의 실동작에 필요한 최소 변경,
  src/ 무변경), (b) 홈서버로 이관했던 사용자의
  **로컬 복귀 절차** 문서화, (c) M5 실복원으로 U-4(Ollama 모델명)·U-5(litellm 우회) 실증.
- **코어 재초점 정합**: 이 슬라이스는 동결 게이트 질문("second-brain 사용자에게 무엇을
  주는가")에 직접 답한다 — 오프라인 동작·Desktop 연결·측정 루프 자연 복원·설치 서사 단순화.

## Problem

1. 홈서버 이관 사용자(M5 포함)가 **로컬로 되돌아오는 절차가 문서에 없다** — 복귀 경로는
   코드에 살아 있으나(F-7·F-8) 절차가 기기 노트·문서 어디에도 정리돼 있지 않다.
2. Ollama 직결(게이트웨이 없는 최소 스택)이 코드상 가능(F-20)하나 **문서화 0** —
   `.env.example`에 `EMBEDDINGS_URL` 자체가 없고, 키 가드(F-19)가 무인증 Ollama UX와 충돌.
3. M5는 현재 두뇌 없음(Ollama 중지·색인 삭제·원격 의존) — 오프라인이면 second-brain 불능,
   Desktop 미등록.
4. 하이브리드 위상(로컬 기본 + 원격 옵션)이 어디에도 선언돼 있지 않아, 045 이후 "원격이
   새 기본"으로 오해될 여지(실제로 M5가 그렇게 운영됨).

## Objective

코드가 이미 보존한 local-first 경로를 **제품의 명시 기본으로 재확정**한다 — (1) 하이브리드
위상(로컬 기본·원격 옵션) 문서 선언, (2) Ollama 직결 구성의 일급 문서화(+키 처리 방식 확정),
(3) 홈서버→로컬 **복귀 절차** 문서화, (4) M5 실복원(도그푸드)으로 절차·미확인(U-4·U-5)을
실증하고 Desktop 등록까지 검증.

## Expected outcome

- 새 사용자: README 로컬 서사 그대로 + Ollama 직결 옵션이 `.env.example`에서 보임.
- 이관 사용자: 문서의 복귀 절차대로 로컬 두뇌를 복원할 수 있다(M5로 실증됨).
- M5: 오프라인에서도 recall/capture 동작, Claude Code(stdio)·Claude Desktop 모두 연결,
  로컬 query-log에 다시 기록 축적(품질 리포트 루프 자연 복원 — 코어 로드맵 슬라이스 2 흡수).
- 홈서버: 옵션으로 존치(오늘 최신화된 상태 그대로) — 위상만 "옵션"으로 명시.

## Success metrics

- [x] M5에서 `whoami`가 로컬 stdio로 응답하고, 원격 MCP를 껐을 때(오프라인 등가)도 recall이
  동작한다 — 도그푸드 관찰(restore-log 단계 ⑤: whoami·search 0.573 회수·capture→로컬
  query-log 41줄).
- [ ] Desktop에서 localmind 도구 목록이 보인다 — **사용자 확인 대기**(config 등록은 AC-5로
  확인 완료, UI 노출은 AI가 관찰 불가 — 사유 기재).
- [x] 복귀 절차 문서가 M5 실행 로그와 1:1로 일치한다 — AC-3(치환 사유 부기 포함, r1 clean).
- [x] U-4·U-5가 실증으로 확정돼 문서에 반영된다 — 재색인 로그(bge-m3 전환·1200파일)·curl
  직결(1024차원), .env.example·복귀 절차에 반영(AC-6).

## Non-goals

- 홈서버 MCP 서비스 은퇴(A안 아님 — 옵션 존치, 서비스·코드 무변경).
- http 전송 모드(specs/045) 코드 변경·제거.
- recall 품질(랭킹·관련성) 개선 자체 — 후속 슬라이스(측정 루프 복원까지가 이번 몫).
- 모바일 접근 경로.
- 다른 기기(M1 등) 복원 — 절차 문서로 재현 가능하게 하는 것까지(실행은 각 기기에서).

## Constraints

- 메타 동결 게이트 준수 — 이 슬라이스의 모든 변경은 second-brain 사용자 가치로 정당화.
- repo 코드 변경 최소(사실 지도 결론 존중) — 키 가드 완화 여부는 spec에서 확정하되 보수
  기본은 코드 무변경(더미 키 문서화).
- M5 복원은 실환경 변경 — 각 단계 관찰·기록, 기존 자산(.env·모델) 보존, 실패 시 원상 복귀
  가능(원격 MCP 등록은 복원 검증 전까지 제거하지 않음 — 병행 후 전환).
- 기기 노트(devices/shaulm5local.md — 사용자 벌트)는 셋업 변경 시 갱신이 명시된 관례이므로
  M5 복원 후 갱신하되 변경 요지를 보고에 명시.

## Stakeholders

단일 사용자(설치한 개인 누구나 — 비개발자 포함). 특히 홈서버 없이 설치하는 신규 사용자와
이관 후 복귀하려는 사용자.

## Risks

- **U-4/U-5가 실증에서 어긋날 가능성**(Ollama 직결이 모델명·스택 문제로 실패) → litellm
  경유 경로(F-6 기본값)가 검증된 fallback — 문서에 두 경로 모두 기재.
- M5 재색인 비용(노트 전량 임베딩) → 1회성, watcher 증분 이후 무비용. 실행 시간 기록.
- 병행 기간의 이중 등록 혼선(localmind-remote + 로컬 localmind) → 이름이 달라 공존 가능,
  검증 후 원격 등록 정리 여부는 사용자 확인.
