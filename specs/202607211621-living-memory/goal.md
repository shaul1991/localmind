# goal — 살아있는 기억 (Phase C: 결정 스키마·세션 주입·재접촉 리마인드)

## Background

product-vision(2026-07-21 확정)이 localmind를 "살아있는 why 저장소"로 재정의했다. 현재의
capture/search는 비정형 노트만 다룬다 — 결정(선택/이유/전제)을 구조로 갖지 않아, "왜 그렇게
정했나"의 소환이 전문 검색에 의존하고, 전제가 낡아도 시스템이 모른다. vision §2의 세 능력
(기억한다·살아있게 유지한다·따라다니게 한다) 중 두 번째·세 번째가 현재 형태에 없다.

## Problem

1. 결정이 비정형 텍스트로 저장돼 **전제(유통기한 있는 층)를 분리 추적할 수 없다** — 낡은
   결정 위에서 작업해도 아무도 모른다 (vision §4·§5).
2. 새 세션이 관련 결정·방향을 **가지고 시작하지 못한다** — 읽기 루프가 pull 모델이라 습관이
   되지 못했음이 실측됨 (vision §6-3).

## Objective

1. 결정을 **선택/이유/전제(휘발성 표시)** 3층 구조로 캡처할 수 있다 — 기존 capture 흐름의
   확장으로, 사람의 추가 작업 없이 (vision §4, §6-1).
2. 새 세션이 한 번의 호출로 **관련 결정·방향 브리핑을 받아 시작**한다 (vision §2-3).
3. 휘발성 높은 전제가 오래 미검증이면, 그 결정이 **다시 조회되는 순간 비차단 신호**로
   알린다 — 검증 자체는 호스트 AI의 몫, localmind는 신호만 (vision §5, §6-2).

## Expected outcome

결정 노트가 구조를 갖고 쌓이기 시작하고, 세션 시작 브리핑(주입)과 전제 낡음 신호가
실동작한다 — vision의 핵심 가치 3종이 최소 형태로 가동된다. (단 vision §5의 "재검증 **이력**
누적(결정의 생애 타임라인)"은 v1 범위 밖 — last_verified 단일 값만, OQ-C3. advisory ⑥)

## Success metrics

- 결정 캡처가 **capture_note 1회 호출**로 완료된다 (사이드이펙트 성립 — 사람 태스크 0).
- 새 세션이 **brief 1회 호출**로 관련 결정 맥락을 확보한다 (도그푸드로 실증).
- 신호 오탐 0: 휘발성 낮은/최근 검증된 전제만 가진 노트에는 신호가 붙지 않는다.
- 기존 비정형 노트의 캡처·검색 동작 100% 비회귀.

## Non-goals

- 기존 노트의 소급 구조화 (forward-only — vision OQ-V4).
- 전제의 라이브 검증 수행 주체가 되는 것 (호스트 AI의 몫 — localmind는 신호만).
- 주기 스캔 자동화 (보조 트리거는 후속 — 이번엔 재접촉 트리거만).
- 검색 스택 변경 (Phase D의 몫).

## Constraints

- **vision §6 비침습 불변식 전부**: 쓰기=사이드이펙트, 알림=비차단, 읽기=필요한 순간 자동,
  밀도=위험 비례. 어기는 형태는 AC에서 결함이다.
- great-reduction(A+B) 이후의 축소된 코드베이스 위에 구현한다 (도구 3개 + 신규).
- 도구 표면 최소 유지: 신규 도구는 brief 1개만 — 결정 캡처는 기존 capture_note 확장.
- 에러 메시지는 비개발자가 이해할 평이한 한국어 (AGENTS.md 오픈소스 대상).

## Stakeholders

단일 사용자 (설치한 개인 누구나 — 비개발자 포함) · 호스트 AI 런타임 (Claude Code·Codex 등).

## Risks

- 스키마가 과해지면 캡처가 무거워져 §6-1 위반 — 최소 3층으로 제한, 확장은 실사용 후.
- 리마인드 신호가 소음이 되면 무시 습관화 — 휘발성 high + 미검증 기간 조건의 보수적 발화.
- great-reduction과의 병렬 작업 시 같은 파일 충돌 — A+B 머지 후 이 슬라이스를 얹는 순서 고정.

## 티어 판정

**Tier 2** — 하드 신호 해당: 신규 도메인 개념(결정 3층·전제 휘발성·브리핑), 계약 변경(MCP
도구 표면에 brief 추가, capture_note 입력 스키마 확장, 노트 frontmatter 직렬화 형식 확장).
