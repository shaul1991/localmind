# goal — 거버넌스 재보정: 확인 루프 완화 · 결과 불변식 전환 · 티어 정교화 (오늘의 최선)

## Background

사용자 문제의식(2026-07-20): "현행 flow는 약한 모델 시절 안정적이었지만, 모델 능력이 오르며
일부 층이 역량을 제한하는 마이너스가 됐다." 이를 deep research(105 에이전트, 주장별 3표 적대
검증, 25 claims — 전문은 이 폴더 `evidence/research-report.md`)로 검증했다. 핵심 발견:

- **완화 지지**: per-action 승인의 93%가 그대로 승인됨(정보 가치 낮음, Anthropic 실측 [T2]) —
  프론티어 랩 수렴 방향은 "건별 승인 → decision-by-exception + 위험 보정 감독". 에이전트는
  인간 개입의 2배 이상 스스로 명확화 질문을 시작한다(강제 밀도를 낮춰도 중요 분기 질문이
  사라지지 않음).
- **유지 지지**: 프론티어 모델도 과신(확신 ~88% vs 정확도 ~79%, RLHF가 증폭)하고 유창함은
  신뢰 신호가 아니다. 격리 검증자만으론 불완전(게이트 우회 36.8% 실측) — 결정적 게이트 병행
  필수. **"제거"를 지지하는 근거는 0** — 모든 출처가 "재배치"를 말한다.
- **동형 확인**: 현행 티어링(위험 비례 의식)은 문헌 수렴 패턴(GAIE 결정적 라우팅 등)과
  구조적으로 동일 — 방향이 맞았고, 정교화(가역성·검증가능성 축)만 남았다.

## Problem

1. **확인 루프 밀도**: 인터뷰 프로토콜의 균일 5단계 강제(모든 결정·구현·다단계 작업에서 재확인
   대기)가 저위험 분기에서도 왕복을 만든다 — 93% 승인률 실측이 보여주는 낮은 정보 가치.
2. **절차 미시규정**: goal-impl 절차 서술이 수단(어떻게)을 세목으로 고정해, 강한 실행자의 더
   나은 경로 재량을 막는다 — 유지해야 할 것은 결과 불변식(무엇이 참이어야 하나)이다.
3. **티어 트리거의 축 누락**: 하드 신호에 가역성·검증가능성 축이 암묵적이라 판정이 흐려질 수
   있다(문헌은 이 두 축을 명시 라우팅 기준으로 씀).
4. **재보정 장치 부재**: 위임 컷포인트는 모델 개선에 따라 이동하는데(전 출처 명시), 거버넌스를
   주기 재보정하는 규칙이 없다 — 이번 같은 일회성 대수술이 반복될 구조.

## Objective

검증 계층(격리 self-review·결정적 게이트·Live-Verify·TDD)은 **문구·의미 무손상으로 유지**한
채 — (1) 확인 루프를 위험 보정형(decision-by-exception)으로 완화하고, (2) goal-impl 절차를
"결과 불변식 1급 + 절차는 권장 기본"으로 재서술하며, (3) 티어 트리거에 가역성·검증가능성 축을
명시하고, (4) 텔레메트리 기반 주기 재보정 규칙을 신설한다.

## Expected outcome

- 저위험 분기에서 AI가 권고안과 근거를 제시하고 **진행하며**, 사용자는 예외에만 개입한다.
  고위험 분기(하드 신호·비가역·범위 변경)는 현행 재확인 강제가 유지된다.
- goal-impl 실행자는 불변식(TDD red 관찰·base freshness·round 예산·PR 게이트 등)을 지키는 한
  수단 경로에 재량을 갖는다 — 약한 실행자를 위한 권장 레시피는 강등된 형태로 보존.
- 티어 판정문에 가역성·검증가능성이 명시 기준으로 들어간다.
- retro 주기마다 self-review 텔레메트리를 근거로 확인 루프·티어 경계의 재보정을 검토한다.

## Success metrics

- 개정된 rules base가 `make rules-deploy`로 전 소비 표면(`~/.claude/localmind-rules.md` 등)에
  드리프트 없이 배포된다.
- 유지 계층의 기존 문구 계약 테스트(localmind repo)가 전부 green — 검증 계층 무손상의 기계 확인.
- 각 개정 rule 파일에 개정 이력(ADR 인라인 — 날짜·근거·출처)이 남고, 결정 노트가 second-brain에
  적재된다.
- 개정 후 첫 Tier 2 실행에서 저위험 분기 왕복 수가 체감 감소(정성 확인)하고, 고위험 분기
  재확인은 유지됨을 사용자가 확인한다. (주의: 이 지표는 반증이 약한 정성 지표다 — 정량화는
  OQ-2의 왕복 텔레메트리 신설 여부에 걸려 있으며, 첫 재보정 retro에서 판단한다.)

## Non-goals

- 격리 self-review·2라운드 상한·결정적 게이트(preflight·EEXIST·base freshness·main push 금지·
  PR)·TDD·Live-Verify의 완화 — **명시적 비대상**(조사가 유지를 지지, 제거 근거 0).
- 렌즈 병렬·critic 캐싱 규약(202607201808 확정)의 변경.
- matrix-as-map vs reasoning-blind 긴장의 해소 — open question으로 기록만(후속 슬라이스).
- 벌트(second-brain) 캐논 직접 개정 — capture(결정 노트)까지만, 승격은 사용자.
- 자동화된 신뢰 점수·티어 자동 하향 메커니즘 구현 — 재보정은 사람이 retro를 보고 결정(조사도
  조작적 정의 부재를 OQ로 남김).

## Constraints

- base 개정은 정본(`~/.localmind/rules/base/`)에서만, 개정 이력(ADR) 동반 — 기존 base 개정 관례.
- 검증 계층 문구·의미 무손상: 기존 계약 테스트 green이 하드 게이트.
- 완화는 "인간 제거"가 아니라 "예외 개입으로 재배치" — 결정 로그·보고 의무는 유지.
- goal-impl 재서술은 약한 실행자 호환을 잃지 않는다(불변식 + 권장 기본의 2층 구조 — §15
  "도구 무관 원칙·권장 기본" 패턴 재사용).
- 데이터 폴더(rules)와 repo(AGENTS.md·SKILL)의 이중 대상 — 커밋·배포 경로가 다름을 plan이 명시.

## Stakeholders

단일 사용자(설치한 개인 누구나 — 비개발자 포함). 전 프로젝트의 AI 에이전트 런타임(Claude
Code·Codex·Gemini CLI)이 이 규칙을 주입받는다.

## Risks

- **완화 근거의 vendor 편향** — 축1·2 핵심 출처가 Anthropic/OpenAI(T2, 이해관계 정렬). 완화 폭을
  보수적으로 잡고(고위험 강제 유지), 재보정 규칙으로 되돌릴 수 있게 설계해 완화.
- **의식 비용 실증 부족(축3)** — 통제 실험 부재. 완화 효과는 자체 텔레메트리로 사후 측정.
- **과완화로 의도 어긋남 재발** — 인터뷰 프로토콜의 존재 이유(답변 직후 다른 해석으로 실행)가
  재발할 수 있음 → 저위험에서도 "이해 요약 제시 + 진행"(요약 생략 불가)으로 어긋남 조기 노출
  유지.
- **이중 대상 드리프트** — rules 정본과 repo 문서가 갈릴 위험 → 배포 파이프라인 실행 + 계약
  테스트로 확인.
