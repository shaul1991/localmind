# 자주 묻는 질문 (FAQ) — 기획자·실사용자 관점

> 실제 사용자(기획자)가 localmind를 처음 검토하며 던진 질문들을 정리했습니다.
> 개념이 처음이면 [비유로 이해하기](concepts.md)부터 보시면 더 쉽습니다.

---

## 1. 개념 — 무엇이고, 왜, 누구에게

### Q. localmind가 정확히 뭔가요?
지금 쓰는 **Claude Desktop / Cursor에 "기억력"과 "내 자료를 읽는 눈"을 달아주는 부품**입니다.
새 앱을 배우는 게 아니라, 쓰던 도구에 부품 하나 끼우는 것. 대화 방식은 그대로이고, AI가 *내 일*에 대해 똑똑해집니다.

### Q. 저는 AI를 가끔만 쓰는데 꼭 필요할까요?
**사용 빈도보다 "들고 있어야 할 맥락의 양"이 기준입니다.**
- 가끔 대화만 하고 그 자리에서 끝난다 → **아직 필요 없음** (Claude Desktop으로 충분)
- **프로젝트를 여러 개** 굴리고, 요구사항·검토 등 *"전에 우리가 뭐라고 했더라"* 가 잦다 → **가치가 큼**

### Q. 어떤 사람에게 효과가 큰가요?
프로젝트 3~4개를 동시에 관리하는 기획자처럼 **맥락 전환·결정 추적·내 문서 검색**이 많은 사람.
한 번의 대화가 아니라 **"파고든 결과가 매번 증발하는"** 손실을 메워 줍니다.

## 2. 노트·기억 — 무엇이고 어떻게 쓰나

### Q. "노트"가 뭔가요?
**컴퓨터에 저장된 글(텍스트 파일)** 일 뿐입니다. localmind는 상자 두 개를 들고 있습니다:

| 상자 | 내용 | 예시 |
|---|---|---|
| 📁 **문서함**(.md 노트) | AI가 대화 중 남긴 기록 + 내가 쓴 문서 파일 | "결제는 OAuth로 결정(이유: …)", `회의록.md` |

> 정본은 언제나 **파일**입니다 — 데이터베이스가 아니라 폴더 속 마크다운.

### Q. 노트는 어떻게 쌓이나요?
- **말해서**: 대화 중 "이거 기록해둬: …" → AI가 노트 파일로 저장(`capture_note`)
- **연결해서**: 노트 폴더(`NOTES_DIR`)를 한 번 가리키면 → 기존 문서 통째로 흡수, 이후 자동

### Q. 쌓인 걸 어떻게 쓰나요?
**그냥 평소처럼 물어보면** 됩니다. AI가 알아서 노트를 검색(`search_notes`)해 근거 있는 답을 줍니다. 별도 조작 없음.

---

## 3. 신뢰·투명성 — AI가 적은 걸 어떻게 확인하나

### Q. AI가 알아서 기록한다는데, 뭘 적었는지 어떻게 믿죠?
**몰래 쌓는 구조가 아닙니다.** 적히는 모든 건 ① 내 컴퓨터 안에, ② 열어볼 수 있는 평문으로 남습니다.

- 📁 **문서함** → 그냥 폴더 열어 파일로 확인·수정·삭제
- ✍️ **적는 순간 화면에 보임** — `capture_note` 호출과 저장 경로가 대화창에 그대로 표시됩니다

> 신뢰를 "믿어주세요"가 아니라 **"직접 확인하세요"** 로 드리는 구조. 전부 로컬, 외부 전송 없음.

## 4. 시작하기 — 설치와 연동

### Q. 처음에 뭘 하면 되나요? (1회, 약 10~15분)
> 아직 "버튼 하나" 앱은 아니지만, `make` 명령이 **한 단계씩 안내**해줍니다.

1. **준비물** — Node.js ≥ 20 + Ollama(임베딩 엔진 — 맥: `brew install ollama`).
2. **받아서 켜기**
   ```bash
   git clone https://github.com/shaul1991/localmind && cd localmind
   make setup   # 준비물 점검 → 설치·빌드 → 임베딩 설정 안내 → 연결 점검까지 한 단계씩 안내
   ```
   별도 상주 서비스가 없습니다 — MCP 서버는 앱이 필요할 때 자동으로 뜹니다(Ollama만 켜두면 끝).
3. **앱에 연결** — Claude Code면 **원클릭** `make mcp-install`. Cursor/Claude Desktop이면 `make mcp-config`가 출력한 JSON을 설정에 붙여넣기.
   ```bash
   make mcp-install                       # 기본 노트 폴더 ~/.localmind
   make mcp-install NOTES_DIR=/내/노트폴더  # 내 .md 노트로 바로 RAG
   ```
4. 앱 재시작 → `whoami`·`capture_note`·`search_notes` 도구가 생김.

이후 매일 하는 것: **그냥 대화.** 설치는 한 번뿐 — 자세한 사용은 👉 [사용법](usage.md).

### Q. Cursor에서도 되나요?
**네.** 방식은 Claude Desktop과 동일, 설정 파일만 `.cursor/mcp.json`(또는 Settings → MCP)로 다를 뿐입니다.

### Q. Cursor·Claude·Codex를 오가도 배경 설명을 다시 안 해도 되나요?
**"저장된" 맥락에 한해 그렇습니다.**

| 종류 | 도구 바꿔도 따라오나 |
|---|---|
| 기억해둔 결정 / 문서함 노트 | ✅ 따라옴 (재설명 불필요) |
| 방금 그 창에서 한 *저장 안 한* 대화 | ❌ 자동 이동 아님 |

→ 비결: **중요한 결론은 "노트로 기록해둬" 한마디로 박아두기.** 그러면 어느 도구든 살아있음.
전제: 도구마다 **같은 `NOTES_DIR`** 로 연결(= 같은 두뇌 공유).

## 5. 운영 — 성능과 백업

### Q. 노트가 아주 커지면(예: 100GB) 느려지나요?
**아니요. 질문마다 전부 읽지 않습니다.** 미리 만든 **색인**으로 관련된 몇 조각에만 곧장 점프합니다(도서관 카탈로그처럼).
- 꺼내오는 양이 노트 크기에 비례해 늘지 않음 → **질문 속도 거의 일정**
- 커지면 늘어나는 건 *질문 속도*가 아니라 **저장 공간 + 쌓을 때 1회 정리 시간**
- 개인 노트로 100GB는 사실상 불가(보통 수백 MB~몇 GB). 순수 텍스트 100GB ≈ 수천만 페이지
- 진짜 한계는 속도보다 **정확도**(많아지면 엉뚱한 조각이 섞일 수 있음) → 튜닝·프로젝트 분리로 관리

### Q. 컴퓨터가 바뀌거나 고장나면 백업은요?
**노트·자산은 `make backup`이 챙기고, 나머지(색인·앱)는 다시 만들면 됨.**

| 대상 | 방법 |
|---|---|
| 📁 문서함(.md) | `make backup`이 내 private git repo로 커밋·푸시(자동은 `make backup-cron`) |
| 🔎 검색 기록(선택) | 기본은 로컬 전용 — 원하면 `BACKUP_QUERY_LOG=1 make backup` (아래 주의 참조) |
| 🗂️ 색인 / ⚙️ 앱 | 백업 불필요(노트에서 재생성 / 다시 clone) |

복구: 새 컴퓨터에 설치 → 노트 복원 → 색인 재생성 → 끝.

> **한 줄 명령으로**: 백업은 `make backup`, 복구는 새 기기에서 `make recover RESTORE_REPO=<내 백업 repo>`.
>
> **기기를 옮기기 전에**: 이전 컴퓨터에서 `make backup`을 한 번 실행하세요 — 백업 이후에 만든 노트·검색 기록은 새 기기로 넘어오지 않습니다(`make recover`가 시작할 때도 알려줘요).
>
> **노트 저장소를 여러 개 연결해 쓰는 경우**: `make recover` 후 노트 저장소를 연결(`make notes-connect`)하고 `make restore`를 한 번 더 실행하면 자산이 올바른 위치로 복원됩니다(recover가 순서를 안내합니다).

### Q. 노트 폴더 설정이 빠진 채 재색인해도 색인이 지워지지 않나요?
**네, 지워지지 않습니다.** 재색인의 삭제 반영은 "이번에 실제로 읽은 폴더" 안에서만 일어납니다:

- 노트 폴더 설정(NOTES_DIR)이 확정되지 않은 상태의 재색인은 **삭제 반영을 전부 보류**하고
  안내만 합니다(색인 추가·갱신은 정상). 설정을 되살리면 다시 임베딩할 필요 없이 그대로 씁니다.
- 등록된 폴더가 열리지 않으면(외장 디스크 미마운트, 아직 clone 전 등) 그 폴더의 색인은
  **보존**되고 경고로 알려줍니다.
- 더 이상 안 쓰는 폴더의 색인이 남아 있으면 재색인 요약이 알려주며,
  `REINDEX_PRUNE_LABELS=<라벨> make reindex`로 직접 정리할 때만 지워집니다.
- 참고: NOTES_DIR를 `.env`·환경변수 어디에도 두지 않고 기본 폴더만 쓰면 `make reindex`의
  삭제 반영이 안전을 위해 보류됩니다 — `make mcp-install`이 NOTES_DIR를 기록해 주면 정상
  반영됩니다(노트 저장·수정 색인은 언제나 동작).

### Q. 검색만 했는데 색인 파일이 계속 다시 써지나요?
아니요 — 노트에 변경(추가·수정·삭제)이 없으면 검색·재색인은 색인 파일을 다시 쓰지
않습니다(무변경 저장 생략). 변경이 있을 때만 기존과 동일하게 저장됩니다. 그리고
`make doctor`가 "지금 설정에 없는(또는 열 수 없는) 폴더의 색인"을 보여주므로, 재색인을
돌리지 않고도 정리 대상 라벨을 확인할 수 있어요.

### Q. 노트 폴더를 다른 위치로 옮기면 어떻게 되나요?
같은 라벨이 다른 폴더를 가리키게 되면(폴더 이동·외장 디스크 경로 변경) 재색인이
"위치가 바뀌었어요"라고 안내하고 **이전 위치의 색인을 보존**합니다 — 실수로 경로가
바뀐 경우(마운트 변동 등) 색인이 날아가지 않게요. 새 위치가 맞으면 안내된
`REINDEX_ADOPT_REBIND=<라벨> make reindex`로 이전 위치 색인을 정리하면 됩니다.
(이 보호는 위치가 한 번 기록된 다음부터 동작해요 — 업데이트 직후 첫 재색인 전에
이미 옮겨둔 폴더는 비교할 기록이 없어 기존 방식대로 정리됩니다.)

### Q. 업데이트했더니 색인이 새 형식으로 바뀐다고 해요 — 다시 색인해야 하나요?
아니요. 색인이 새 형식(벡터 분리 저장 — 파일이 훨씬 작고 빨라짐)으로 자동 전환되며,
**기존 벡터를 그대로 재사용하므로 재색인이 필요 없습니다.** 알아둘 것 두 가지:

- 업데이트 후 Claude Desktop/Code 등 MCP 클라이언트를 **재시작**해 주세요 — 구버전과
  신버전이 같은 색인을 번갈아 쓰면 서로 형식을 되돌리며 낭비가 생길 수 있어요.
- 구버전으로 되돌아가면(롤백) 색인은 자동으로 처음부터 다시 만들어집니다 — 노트는
  그대로이고 시간만 듭니다. 색인 폴더의 `.brain-index.json.vec-*` 파일은 새 형식의
  벡터 파일이니 지우지 마세요(안 쓰게 되면 지워도 무해).

### Q. 대량 재색인이 너무 오래 걸려요
색인 진행 저장은 기본 10초 간격으로 스로틀됩니다(`BRAIN_SAVE_INTERVAL`초, `0`이면 매
배치 저장 — 노트가 아주 많을 때 저장이 병목이 되지 않게). 임베딩을 호스트 GPU(Metal
등)로 돌리는 구성이면 `make reindex`가 배치 크기(`BRAIN_BATCH`)를 자동으로 32로 올려
줍니다 — 직접 지정하면 그 값이 우선합니다. 엔진 전환은 `make doctor` 안내를 따르세요.

### Q. 검색 기록(쿼리 로그)도 기기 간에 옮기고 싶어요
**`BACKUP_QUERY_LOG=1`로 켜면 됩니다(기본은 로컬 전용).** 백업 저장소에 기기별 파일(`query-log.<기기이름>.jsonl`)로 커밋되고, 다른 기기에서 `make restore` 시 로컬 로그에 중복 없이 병합됩니다(30일 지난 항목은 안 들어옴). 검색 품질 리포트(`make query-report`)의 데이터가 기기 간에 모입니다.

- ⚠️ **검색어 원문이 git 이력에 남습니다** — 한 번 커밋되면 나중에 설정을 꺼도 이력에서 지워지지 않아요. 백업 저장소가 Private인지 확인하고 켜세요(처음 켤 때 백업이 한 번 더 알려줍니다).

### Q. GitHub만으로 백업되나요?
**네.** 노트·기억 둘 다 텍스트 파일이라 git에 딱 맞고, **변경 이력**까지 남습니다. 단:
- ⚠️ **반드시 Private 저장소** (기획 내용이 공개되면 안 됨)
- ⚠️ GitHub는 **자동이 아님** → `git push` 필요 (주기적으로 하거나 자동 예약)

### Q. 노트/기억 말고 개인 설정 파일도 같이 백업하고 싶어요(예: 다른 도구의 전역 설정)
**`BACKUP_EXTRA_FILES`로 지정하면 됩니다.** 기본은 아무 파일도 다루지 않습니다(선택 사항).

```bash
make backup BACKUP_EXTRA_FILES="~/.claude/CLAUDE.md,~/.codex/config.toml"
```

지정한 파일만 백업 저장소(`extras/` 폴더)에 함께 커밋되고, 새 기기에서 `make restore`/
`make recover` 시 원래 위치로 복원됩니다(기존 파일과 다르면 `.bak-날짜시간`으로 보존 후 교체).

- ⚠️ **백업 저장소가 Private이어도 파일 내용은 그대로 커밋됩니다** — 비밀번호·API 키 등
  민감정보가 담긴 파일은 지정하지 마세요.
- `$HOME`(내 홈 폴더) 하위 파일만 지원합니다.

## 6. 안전

### Q. 여러 사람이 쓰는 컴퓨터에서 써도 안전한가요? (공유 머신)
**네.** localmind는 상주 네트워크 서비스가 없습니다 — MCP는 stdio(프로세스 직결)로만 동작하고,
외부 호출은 임베딩(기본: 같은 컴퓨터의 Ollama)뿐입니다. 노트는 내 계정의 파일 권한을 따릅니다
(`.env`는 소유자 전용 600). 원격(http) 모드를 직접 켤 때만 토큰 인증(`MCP_AUTH_TOKEN`)이 강제됩니다.

### Q. AI가 실수로(또는 악성 노트 지시로) 노트를 지울 수 있나요?
**아니요 — AI에게 삭제 도구 자체가 없습니다**(도구는 기록·검색·브리핑·식별 4종뿐 — 전부
쓰기 추가 또는 읽기 전용). 노트 삭제는
사람이 파일시스템/에디터에서 직접 합니다. 과거 휴지통(`.trash/`)에 남은 노트는
`make trash-list`로 확인, 완전 비우기(비가역)는 `make trash-empty`(사람이 직접, 확인 후).
git 백업(`make backup`)을 켜뒀다면 삭제 이력도 git으로 복구할 수 있습니다.
