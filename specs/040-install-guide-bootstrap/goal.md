# Goal: 설치 전 시각적 설치 가이드 — 무의존 부트스트랩 서버

## Background
localmind는 비개발자 포함 오픈소스다. 현재 설치는 `make setup`(터미널 단계별 안내)로만
안내되고, `make ui`(모니터링 웹)는 `npm run ui`(tsx)라 **node_modules가 있어야** 뜬다 —
그 node_modules를 까는 게 `make setup`이라, **fresh clone(설치 전)에는 웹 가이드를 볼 수 없다**
(부트스트랩 역설, 2026-07-07 확인).

## Problem
비개발자에게 **터미널만으로 하는 설치는 허들이 높다**. "터미널을 어떻게 여나", "Docker가
뭔가", "지금 준비물이 됐나", "이 다음에 뭘 치나"에서 막힌다. 그러나 설치 전에는 앱(웹 UI)이
아직 없어서 시각적 안내를 줄 수단이 없다.

## Objective
**설치를 시작하기 전에도** 브라우저로 열 수 있는 **아주 상세한 시각적 설치 가이드**를 제공한다.
의존성 설치 없이(=Node 내장 모듈만으로) 도는 작은 부트스트랩 서버가 (1) **준비물을 실시간
점검**해 무엇이 됐고 안 됐는지 배지로 보여주고, (2) **각 설치 단계를 비개발자도 따라 할 만큼
자세히** 설명하며 명령을 **복사 버튼**으로 제공한다. 웹은 **명령을 실행하지 않는다**(복사만).

## Expected outcome
- 사용자가 `git clone` 직후 **한 명령(`make guide`)으로 브라우저 가이드**를 열고, 준비물 상태를
  초록/빨강으로 확인하며, 각 단계 명령을 복사해 터미널에 붙여넣어 설치를 완주한다.
- 다른 터미널에서 `make setup`을 도는 동안 가이드를 **새로고침하면 체크가 초록으로** 바뀌어 진행 확인.

## Success metrics
- [x] 비개발자가 가이드만 보고 준비물 점검 → `make setup` 흐름을 따라감. ✔ 브라우저 dogfood(준비물 배지·설치 3단계·복사·문제해결 렌더)
- [x] 부트스트랩 서버가 **node_modules·dist·tsx 없이** 실행(무의존 불변). ✔ import 스캔 + 심링크 서브프로세스 회귀 테스트 + 격리 폴더 실증
- [x] **명령 실행 엔드포인트 0** — GET만. ✔ POST/PUT/DELETE 405 테스트
- [x] 준비물 점검이 setup.sh preflight와 정합(docker/.env/ollama 일치, node는 guide가 더 엄격). ✔ 스키마 + 로직 리뷰

## Non-goals
- 브라우저에서 `make setup`·설치 명령을 **실행**하는 것(복사만 — 2026-07-07 결정).
- 설치 **후** 라이브 설정 관리 — 그건 specs/039(`make ui` 설정 페이지)가 담당(생애주기 분담).
- Node 자체 설치 안내의 자동화(Node 없으면 가이드도 못 뜸 → README 폴백 안내).
- 다국어(1차 한국어).

## Constraints
- **무의존 불변**: 부트스트랩 서버는 **Node 내장 모듈만**(http·fs·child_process·os·path) 사용.
  `npm install`·`npm run build`·tsx·외부 패키지 금지 — 설치 전 실행이 목적.
- **읽기 전용/복사만**: HTTP는 GET만. 점검은 읽기 전용 명령(`docker info`·`node -v` 등)만 spawn,
  상태 변경·설치 실행 없음. 시크릿 노출 없음(설치 전이라 시크릿도 없음).
- **로컬 전용**: `127.0.0.1` 바인딩만.
- **전제**: Node가 설치돼 있어야 실행(문서상 준비물). 미설치 시 README 폴백.

## Stakeholders
단일 사용자(localmind를 설치하려는 개인 누구나 — **특히 비개발자·터미널 초심자**).

## Risks
- **Node 미설치**: 가이드도 Node로 도니 못 뜸 → `make guide`가 Node 부재를 감지해 친절히 README로 유도.
- **점검 오탐**: Docker/ollama 감지가 환경차로 틀릴 수 있음 → 판정 불가는 "확인 불가"로 안전 저하.
- **과설계**: 비개발자용이라 상세해야 하지만, 서버는 최소(단일 .mjs)로 — 상세함은 콘텐츠(HTML)로, 복잡도는 낮게.
- **포트 충돌**: 8799 사용 중이면 안내 후 대체 포트.
