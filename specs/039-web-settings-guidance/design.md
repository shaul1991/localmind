# Design: 웹 설정 페이지 — 상태·복사·안내 (디자인 사전 정의)

> 이 문서가 확인되기 전에는 UI 구현에 착수하지 않는다(AGENTS.md 디자인 게이트).
> **정본은 기존 034 디자인 시스템**(`public/ui/style.css`) — 여기서는 그 토큰·클래스를
> **재사용**하고, 이 화면에 필요한 최소 신규(복사 버튼)만 정의한다.

## 0. 페이지 구성 (설정 페이지 전체 레이아웃)

기존 "설정(읽기 전용)" 페이지의 골격은 유지하고, **각 설정 행에 [상태 배지]와 [명령+복사]를
추가**한다. 최상단에 3단계 안내 카드를 둔다.

```
┌─ 설정 ────────────────────────────────────────────────┐
│                                                        │
│ ┌ 안내 카드 (card) ────────────────────────────────┐  │
│ │ ① 바꿀 항목의 [복사]를 누르고                     │  │
│ │ ② 터미널에 붙여넣어 실행한 뒤                      │  │
│ │ ③ 이 페이지를 [↻ 새로고침] 하세요                 │  │
│ └──────────────────────────────────────────────────┘  │
│                                                        │
│ 연결 상태                                    [↻ 새로고침]│
│ ┌ card ────────────────────────────────────────────┐  │
│ │ [● 됨]     Claude 구독 인증                        │  │
│ │            └ 바꾸려면: `make claude-token`  [복사] │  │
│ │ ─────────────────────────────────────────────────  │  │
│ │ [● 안됨]   Claude Desktop MCP 연결                 │  │
│ │            └ `make mcp-desktop`             [복사] │  │
│ │ ─────────────────────────────────────────────────  │  │
│ │ [○ 확인 불가] Claude Code MCP 연결                 │  │
│ │            └ `make mcp-install`             [복사] │  │
│ │ ─────────────────────────────────────────────────  │  │
│ │ [● 됨]     노트 폴더  ~/.localmind                 │  │
│ │            └ 바꾸려면 .env의 NOTES_DIR 수정  [복사] │  │
│ └──────────────────────────────────────────────────┘  │
│                                                        │
│ 설정 값 (.env — 읽기 전용, 시크릿은 서버에서 가려짐)     │
│ ┌ card ── (기존 그대로 유지) ──────────────────────┐  │
│ │ LOCALMIND_API_KEY = ••••••  · CLAUDE_… = ••••••   │  │
│ └──────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────┘
```

- 배지 상태 3종: `● 됨`(ok, 초록) · `● 안됨`(missing, 주황) · `○ 확인 불가`(unknown, 회색).
- 기존 하단 ".env 마스킹 값" 카드는 **변경 없이 유지**(외과적 변경).

## 1. 디자인 시스템 패턴
- **패턴**: 기존 **카드 + 설정 행(row)** 유지. 각 행 = `[배지] [라벨/값] [명령 chip + 복사 버튼]`.
  최상단 **안내 카드**로 흐름(복사→붙여넣기→새로고침)을 명시.
- **근거**: 034와의 **일관성**(사용자가 이미 아는 화면) + **상태 가시성**(배지로 "됐나"를 즉시).
  미적 완성도보다 **상태 가시성·추적성 우선**(AGENTS.md 디자인 기본 원칙).
- **기각한 대안**:
  - *설정 마법사(단계별 모달)* — 과설계. 읽기 전용 안내에 불필요한 상태 관리.
  - *행 클릭 시 웹에서 실행* — 보안 경계 위반(Tier B/C, 이번 non-goal).

## 2. 디자인 토큰 (요소별 적용 — 값은 기존 정본 참조)

**재사용(신규 아님)** — `style.css :root`에 이미 존재. 이 화면에서의 **적용처**를 못 박는다:

| 토큰 | 값(라이트 / 다크) | 이 화면에서의 적용처 |
|---|---|---|
| `--color-ok` | `#15803d` / `#22c55e` | 배지 `ok`("됨") 배경 |
| `--color-warn` | `#b45309` / `#f59e0b` | 배지 `missing`("안됨") 배경 |
| `--color-idle` | `#475569` / `#94a3b8` | 배지 `unknown`("확인 불가") 배경 |
| `--color-on-state` | `#fff` / `#0f172a` | 배지 글자색 |
| `--color-surface` | `#fff` / `#1e293b` | 카드·복사 버튼 배경 |
| `--color-border` | `#e2e8f0` / `#334155` | 카드 테두리·행 구분선·복사 버튼 테두리 |
| `--color-bg` | `#f8fafc` / `#0f172a` | 명령 chip 배경(코드 느낌) |

| 토큰 | 값 | 용도 |
|---|---|---|
| `--copy-feedback-ms` | `1500ms`(JS 상수) | "복사됨" 표시 지속 시간 |

## 3. 컴포넌트 정의

### 3.1 상태 배지 (기존 `.badge` 재사용)
- **목적**: 한 설정이 지금 어떤 상태인지 색+문구로.
- **변형**: 없음(단일). 문구만 상태별로.
- **상태 · 표시**(치수: 높이 20px, 패딩 `2px 8px`, radius `999px`, `--font-caption`, 글자 `--color-on-state`):
  | 상태 | 클래스 | 배경 | 문구 |
  |---|---|---|---|
  | ok | `.badge.ok` | `--color-ok` | `됨` / `연결됨` |
  | missing | `.badge.warn` | `--color-warn` | `안됨` / `미설정` |
  | unknown | `.badge.idle` | `--color-idle` | `확인 불가` |
- **상태 전이**: 페이지 진입 → `/connections` 조회 전에는 `unknown`("확인 중"으로 문구만 대체) →
  응답 도착 시 각 항목 상태로 확정. **조회 실패 시**(UI 서버 다운 등)는 배지를 침묵으로 unknown 처리하지
  않고 034 `card()` 패턴대로 **명시적 error-state 카드**("불러오지 못했어요 …")를 보인다 — 서버 다운을
  숨기지 않는 게 더 정직(구현 반영, self-review 경미-2).

### 3.2 명령 복사 버튼 `.copy-btn` (신규)
- **목적**: 실행할 명령을 클립보드로 한 번에.
- **치수**: 높이 `28px`, 패딩 `0 var(--space-sm)`, radius `6px`, 테두리 `1px solid var(--color-border)`,
  배경 `--color-surface`, 글자 `--font-caption`, `cursor: pointer`. 좌측 명령 chip과 gap `--space-sm`.
- **변형**: `default`(텍스트 "복사") · 폴백 시 chip이 `user-select:text`로 선택 가능.
- **상태 · 표시**:
  | 상태 | 표시 | 전이 조건 |
  |---|---|---|
  | idle | 테두리 border·글자 text·"복사" | 기본 |
  | hover/focus | 테두리 `--color-primary` | 마우스/키보드 포커스 |
  | copied | 배경 `--color-ok`·글자 `--color-on-state`·"복사됨 ✓" | 클릭 성공 → `--copy-feedback-ms` 후 idle 복귀 |
  | 폴백(클립보드 불가) | 명령 chip을 프로그램적으로 선택 + 버튼 문구 "드래그해 복사" | `navigator.clipboard` 미가용(구현 반영, 경미-3) |
- **상태 전이**: `클릭 → navigator.clipboard.writeText(cmd)` 성공 → `copied`(1.5s) → `idle`.
  실패/미가용 → 폴백 표시(에러 아님). *localhost(127.0.0.1)는 보안 컨텍스트라 clipboard 사용 가능 —
  폴백은 드문 환경 대비.*

### 3.3 설정 행 `.setting-row` (신규 레이아웃, 토큰만 사용)
- **목적**: 배지·라벨·명령을 한 줄에 정렬.
- **치수/레이아웃**: `display:flex; align-items:center; gap:var(--space-md);`
  상하 패딩 `var(--space-sm)`, 행 사이 구분선 `1px solid var(--color-border)`. 최소 높이 `44px`(터치 타깃).
  구성: `[배지(고정폭 auto)] [라벨/값(flex:1)] [명령 chip + 복사(auto, 우측)]`.
- **상태**: 값 없음(예: NOTES_DIR 미설정) → 값 자리에 `--color-text-dim`로 "미설정".

### 3.4 안내 카드 `.guide-card` (기존 `.card` 재사용)
- **목적**: "복사 → 붙여넣기 → 새로고침" 3단계 흐름 안내.
- **표시**: 기존 `.card`(surface·border·radius-card·패딩 md). 번호 목록 3개, `--font-body`.
  우상단 `[↻ 새로고침]`은 텍스트 버튼(글자 `--color-primary`) — `/connections` 재조회 + 배지 갱신.
- **상태**: 새로고침 클릭 → 버튼 문구 "확인 중…"(비활성) + 카드 skeleton → 갱신 완료(구현 반영).

## 4. 반응형
- **기본(≥480px)**: 행은 한 줄. `[배지] [라벨 flex:1] [명령+복사 우측]`.
- **좁은 화면(<480px)**: `.setting-row`가 `flex-wrap:wrap` — 1행 `[배지][라벨]`, 2행 `[명령 chip + 복사]`
  (명령이 라벨 아래로 내려와 가로 스크롤 없음). 명령 chip은 `overflow-x:auto`로 자체 스크롤.
- 페이지 컨테이너 폭·사이드바 등 골격은 034 기존 반응형 그대로(신규 미디어쿼리는 `.setting-row`만).
- 다크 모드: 모든 색은 위 토큰(라이트/다크 자동)만 사용 → 별도 처리 불필요.

## 5. 접근성 기본
- 배지는 색만이 아니라 **문구**로도 상태 구분(색맹 대비) — "됨/안됨/확인 불가".
- 복사 버튼: `<button>` 요소, 키보드 포커스 링(hover와 동일 `--color-primary`), `aria-label="명령 복사"`.
- "복사됨" 피드백은 `aria-live="polite"`로 스크린리더에 통지.

## 6. 에이전트 실행 프롬프트

### worker(구현)에게
```
specs/039-web-settings-guidance의 spec.md·design.md를 기준으로 구현하라.
- 상태색·배지는 기존 style.css의 --color-ok/warn/idle 토큰과 .badge.ok/.warn/.idle 클래스를
  재사용한다(새 색 하드코딩 금지). 새 CSS는 .copy-btn과 .setting-row 레이아웃만 추가.
- 상태 판정은 순수 함수 computeConnections(주입된 입력)로 하고, 파일 IO 실패는 예외가 아니라
  'unknown'으로 흡수한다(AC-3). 시크릿은 값이 아니라 존재 여부만 다룬다(AC-8).
- UI 라우터에는 GET만 추가한다(AC-7 — 쓰기 엔드포인트 금지).
- 복사 버튼: navigator.clipboard 성공 시 'copied'(1.5s)→idle, 미가용 시 텍스트 선택 폴백(AC-6).
- 상태 전이(idle→copied→idle, 진입시 unknown→확정)를 design.md 3장대로 구현하라.
```

### ux-reviewer(점검)에게
```
구현된 설정 페이지를 이 design.md와 대조하라.
- 토큰 위반: 배지·복사 버튼·행이 정의된 토큰(--color-ok/warn/idle, space, radius, font)을
  쓰는가, 색을 하드코딩하지 않았는가.
- 상태 가시성: ok/missing/unknown 3상태가 색+문구로 구분되는가, 조회 중/실패가 표면화되는가.
- 접근성: 배지가 색만으로 상태를 전달하지 않는가, 복사 버튼이 button+aria-label+포커스 링을 갖는가,
  "복사됨"이 aria-live로 통지되는가.
- 반응형: <480px에서 명령이 줄바꿈되어 가로 스크롤이 생기지 않는가.
- 경계: 웹에서 설정을 '쓰는' UI가 없는가(읽기 전용·안내만).
최종 품질 판정은 크리틱(다운시프트 금지).
```
