# Goal: Local Security Hardening (로컬 공격 표면 축소)

## Background — 배경

2026-07-01 품질·보안 평가에서 **보통(M2, M3)** 으로 지목된 두 발견을 다룬다. localmind의
위협 모델은 "내 머신에서 나 혼자"이고 루프백 바인딩으로 LAN은 이미 차단돼 있다. 그러나
루프백 안쪽에도 공격 표면이 남는다:

- **M2 — 같은 머신의 다른 주체**: `LOCALMIND_API_KEY` 미설정(기본값)이면 같은 머신의
  모든 프로세스가 무인증으로 사용자의 Claude/Codex 구독을 소비할 수 있다. 또 Host 헤더
  검증이 없어, 악성 웹사이트가 DNS rebinding으로 브라우저를 경유해 로컬 서버에 요청을
  보내는 고전 공격 경로가 이론상 열려 있다(express.json이 application/json만 파싱해
  실효성은 낮지만, 방어는 한 줄이다).
- **M3 — 간접 프롬프트 인젝션 + 비가역 도구**: `ask_brain`은 노트 본문을 그대로 LLM
  프롬프트에 주입하는데, 같은 MCP 서버에 `delete_note`(즉시 영구 삭제)가 있다. 외부에서
  받아 저장한 노트에 악성 지시가 심어지면 에이전트가 사용자의 노트를 지울 수 있는 구조다.

## Problem — 문제

- 공유 머신·회사 노트북 등에서는 "루프백 = 나 혼자"라는 가정이 깨지는데, 기본 설정이
  이 경우를 안내하지 않는다.
- Host 헤더 미검증은 비용 0의 방어(한 줄)를 하지 않고 있는 상태다.
- `delete_note`는 유일하게 비가역적인 MCP 도구인데 완충 장치(휴지통·확인)가 없다 —
  프롬프트 인젝션이 아니어도 모델의 단순 착오로 노트를 잃을 수 있다.

## Objective — 목표

루프백 안쪽의 잔여 공격 표면 두 가지를 줄인다: (1) HTTP 서버에 Host 헤더 검증을 추가하고
공유 머신 시나리오의 API 키 안내를 강화한다. (2) `delete_note`를 즉시 삭제에서 복구
가능한 휴지통(soft-delete) 방식으로 바꿔, 인젝션·착오에 의한 데이터 손실을 비가역에서
가역으로 낮춘다.

## Expected outcome — 기대 결과

- localhost/127.0.0.1 계열이 아닌 Host 헤더의 요청은 거부된다(DNS rebinding 차단).
- README/FAQ에 "공유 머신이면 `LOCALMIND_API_KEY`를 설정하라"는 안내가 명확히 들어간다.
- `delete_note`가 노트를 영구 삭제하는 대신 휴지통 폴더로 이동시키고, 응답에 복구 방법을
  안내한다. 휴지통 비우기는 사람이 명시적으로 실행하는 명령으로만 가능하다.

## Success metrics — 성공 지표

- 위조 Host 헤더 요청이 403으로 거부되고, 정상(localhost) 요청은 기존과 동일하게 동작한다.
- `delete_note` 후 노트 파일이 휴지통에 존재하고, 검색 인덱스에서는 제외된다.
- 휴지통의 노트를 원위치로 복구하면 재인덱싱 후 검색에 다시 잡힌다.

## Non-goals — 비목표

- 프롬프트 인젝션의 근본 차단(입력 필터링·모델 가드레일)은 범위 밖 — RAG 고유 한계이며,
  이번 범위는 "당하더라도 비가역 피해가 없게"까지다.
- MCP 도구 호출에 인간 승인(HITL) 단계를 넣는 것은 범위 밖 — MCP 호스트(클라이언트)의
  권한 모델 영역이다.
- CORS 헤더 추가는 범위 밖 — 브라우저용 API가 아니므로 CORS를 여는 것 자체가 불필요.
- 상수시간 API 키 비교 등 낮음 등급 항목은 범위 밖(위협 모델상 실익 없음).

## Constraints — 제약

- Host 검증은 기본 활성이되, 리버스 프록시 등 특수 구성 사용자를 위해 환경변수로 끌 수
  있어야 한다(비개발자 기본값은 안전, 고급 사용자는 선택).
- 휴지통은 노트 폴더 내부의 숨김 폴더로 — 인덱싱 대상에서 자연 제외(기존 숨김 폴더 제외
  로직 재사용)되고, 백업(`make backup`)에는 포함되지 않아야 한다(.gitignore).
- `delete_note`의 MCP 도구 시그니처(입력 스키마)는 유지 — 동작만 soft-delete로.
- 에러·응답 메시지는 비개발자가 이해할 수 있는 평이한 한국어로.

## Stakeholders — 이해관계자

- 단일 사용자(설치한 개인 누구나 — 비개발자 포함) — 특히 공유 머신 사용자와,
  외부 콘텐츠를 노트로 저장하는 사용자

## Risks — 리스크

- Host 검증이 정당한 구성(커스텀 hosts 별칭, 컨테이너 내부 호출 등)을 오차단할 수 있음 —
  compose 내부 서비스 간 호출(litellm→localmind:8787)은 Host가 `localmind:8787`이므로
  **허용 목록에 컨테이너 서비스명을 포함해야 함**(놓치면 게이트웨이 전체가 깨짐).
- 휴지통 누적으로 디스크 증가 — 비우기 명령과 함께 오래된 항목 안내를 제공해 완화.
- soft-delete로 바뀌면 "완전히 지웠다"고 믿는 사용자와의 기대 불일치 — 도구 응답에
  휴지통 위치와 완전 삭제 방법을 명시해 완화.
